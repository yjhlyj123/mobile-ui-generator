# Codex 适配器：Pencil Prompt 规则

页面复杂时，先整理结构化 Pencil prompt，而不是把用户原始需求原样传入。

## Prompt 必须包含

- 页面类型与业务目标
- 目标用户与业务场景
- 核心模块及其优先级
- 页面风格约束
- 技术交付约束
- 内容克制规则
- 用户提供了截图、`.pen` 文件或设计草稿时，作为参考信息附上

## 风格约束

- B 端企业基调
- 理性蓝灰方向
- 信息克制
- 弱装饰
- 无营销式布局或文案

## 反 AI 生成感约束

- 布局节奏变化：禁止全页等分网格（2×2、3×3 完美对称），区块应有大小主次之分
- 首屏视觉重心：必须有一个明确的"主角区块"，不是均匀分布
- 内容真实感：使用具体业务数据（"¥125,000"、"张三"、"采购合同-2024001"），不用"标题文本"、"描述信息"
- 排版层次：标题与正文字号差 ≥ 4px，粗细对比明确
- 间距变化：不同优先级区块间距不同，不是统一间距
- 色彩：大面积灰白 + 局部品牌色点睛，不要处处有色彩
- 禁止 AI 套路：不做"渐变头部 + 问候语 + 2×2 金刚区 + 列表"的万能首页模板
- 禁止装饰性半透明圆形/方形散落在头部背景
- 禁止每个图标配不同颜色的圆形底色（彩虹图标）

## 技术约束

- uni-app
- Vue 3
- UnoCSS
- wot-design-uni

## 内容约束

- 不写弱价值描述
- 不自动补充数量提示
- 不写通用功能说明
- 不写宣传式文案

## 重构模式 Prompt 规则

重构模式激活时，为 **3 个方向分别撰写独立的结构化 prompt**。

每个方向的 prompt 必须：

- 在开头标注方向名称与策略
- 从参考页中只提取模块、数据字段、状态和操作——不得复制其布局或视觉处理方式
- 明确描述本方向与参考页面的差异点
- 包含「避免复现参考页的以下元素」段落

```md
# 方向名称：[方向名]
- 方向策略：[一句话说明该方向的核心差异]
- 从参考页提取的内容（仅模块/数据/状态/操作）：
- 本方向的布局策略：
- 本方向的信息层级：
- 避免复现的参考页元素：
- 风格约束：B 端企业、理性蓝灰、信息克制、弱装饰
- 技术约束：uni-app / Vue 3 / UnoCSS / wot-design-uni
```

## 回退规则

- 如果 Pencil MCP 不可用，不得继续生成页面实现代码；必须先输出诊断结果、失败原因与阻塞说明，等待用户确认后再决定是否绕过 Pencil
