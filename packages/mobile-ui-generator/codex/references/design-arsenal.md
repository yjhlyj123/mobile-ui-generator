# 设计弹药库（Design Arsenal）

> 本文件是 AI 把「一句话需求」翻译成 **C 级视觉指令型 Pencil prompt** 的弹药库。
>
> - `design-spec.md` 是**规范底座**：定义有哪些 Token、组件怎么用、平台约束是什么。
> - **本文件是设计素养**：教 AI **怎么把 Token 组合成有层次、有焦点、有节奏的设计**，让 AI 敢写、写得出「主角卡占屏 60% + 金额用 `$font-size-hero(26px)` 做焦点」这种具体决策。
>
> **铁律（贯穿全文）**：凡 `design-spec.md §1` 有 Token 的视觉属性（字号 / 字重 / 间距 / 圆角 / 阴影 / 色彩），一律 **`$token名(数值)` 双标注**，如 `$font-size-lg(18px) + $font-weight-bold(600)`。这样 Pencil 能照值画、代码实现能直接映射 Token 不硬编码。布局尺寸（区块高度、占屏比、装饰尺寸）无 Token，用 `rpx` 或百分比描述。

---

## §0 什么是 C 级视觉指令（标准定义）

同一个需求，prompt 可以写到三种详细度。**本系统只接受 C 级。**

| 级别 | 特征 | 后果 |
|------|------|------|
| **A 骨架型** | 只列「模块1/模块2 + 禁止项」 | Pencil 自由发挥，出图靠运气 |
| **B 内容型** | 真实业务数据填满每个区块，但「长什么样」没说 | 内容到位，视觉仍由 Pencil 猜 |
| **C 视觉指令型** ✅ | 内容 + **视觉层次 + 焦点 + 间距节奏全部锁死** | Pencil 几乎只需照着画，设计灵魂由 prompt 决定 |

**C 级的四个锁死维度：**

1. **内容**：用真实业务数据（`采购合同-2024089`、`¥125,000`、`王磊·采购部`），不用「标题文本」。
2. **视觉层次**：每个区块标注内部元素的字号档 + 字重档 + 色彩 Token（谁大谁小、谁粗谁细、谁深谁浅）。
3. **焦点**：明确指出首屏视觉重心区块（占屏比例）+ 区块内的视觉焦点（哪个数字/状态最跳）。
4. **间距节奏**：区块间距用疏密交替的 Token 档位（不是均匀），制造呼吸。

> **微型对照**（待办列表项）：
> - B 级：`采购合同-2024089 / 王磊·采购部 / ¥125,000 / 待审批`
> - C 级：`左竖色条($status-warning-main 待审橙) | 主标题"采购合同-2024089" $font-size-md(16px)+$font-weight-medium(500)+$color-text-primary | 副信息"王磊·采购部·2h前" $font-size-sm(12px)+$color-text-secondary | 右侧金额"¥125,000" $font-size-lg(18px)+$font-weight-bold(600)+$color-primary（焦点）| 状态标签用四层状态色 warning`

## §1 视觉语言词汇表 —— 把 Token 组合成设计

这一节是核心。每个「设计动作」给出**原理 + 配方 + 正例 / 反例**。写 prompt 时按动作取用。

### 1.1 制造层次（字号 × 字重对比）

**原理**：同一卡片内，靠字号跨档 + 字重对比拉开主次。**标题与正文字号至少跨 2 档**（差 ≥ 4px），否则视觉发灰、AI 感重。

**字号档**（来自 `design-spec.md §1.5`）：`$font-size-xs(10) / sm(12) / base(14) / md(16) / lg(18) / xl(20) / hero(26)`
**字重档**：`$font-weight-regular(400) / medium(500) / bold(600) / heavy(700) / black(800)`

**配方**：

| 角色 | 字号 | 字重 | 色彩 |
|------|------|------|------|
| 页面/区块大标题 | `$font-size-lg(18)`~`$font-size-xl(20)` | `$font-weight-bold(600)`~`heavy(700)` | `$color-text-primary` |
| 卡片标题 / 列表主标题 | `$font-size-md(16)` | `$font-weight-medium(500)` | `$color-text-primary` |
| 正文 | `$font-size-base(14)` | `$font-weight-regular(400)` | `$color-text-regular` |
| 副信息 / 辅助说明 | `$font-size-sm(12)` | `$font-weight-regular(400)` | `$color-text-secondary` |
| 关键指标 / 金额（焦点）| `$font-size-hero(26)` | `$font-weight-heavy(700)` | `$color-primary` 或状态色 |

- ✅ 卡片内最多 3 级字号（标题 / 正文 / 辅助）。
- ❌ 一张卡片出现 4-5 种字号；标题正文字号只差 2px。

### 1.2 制造焦点（让关键信息跳出来）

**原理**：每屏、每个核心卡片都要有一个「最跳」的元素。焦点靠 **大字号 + 重字重 + 品牌色/状态色** 三件套同时叠加。

**配方**：
- 金额、核心指标数值 → `$font-size-hero(26px) + $font-weight-heavy(700) + $color-primary`，加 `¥` 前缀 + 千分位。
- 待处理数量、超时告警 → 数值用状态色（如超时用 `$status-error-main`）。
- 其余信息保持克制（`$font-size-sm/base` + 中性色），**衬托**焦点。

- ✅ 一屏 1 个主焦点 + 每卡 1 个次焦点。
- ❌ 处处加粗加色、人人都想跳 → 等于没有焦点。

### 1.3 制造节奏（间距的疏密交替）

**原理**：区块间距**不均匀**才有呼吸感。相关内容贴近（小间距），不同模块拉开（大间距）。

**间距档**（来自 `design-spec.md §1.6`）：`$spacing-xxs(4) / xs(8) / sm(12) / base(16) / md(20) / lg(24) / xl(32) / xxl(40)`

**配方**：
- 卡片**内**元素间距：`$spacing-xs(8)`~`$spacing-base(16)`
- 列表项之间：`$spacing-sm(12)`
- **模块之间**（区块分隔）：`$spacing-lg(24)`~`$spacing-xl(32)`，关键分界可达 `$spacing-xxl(40)`
- 页面顶部 / 底部留白：`$spacing-xxl(40)`

- ✅ 从上到下间距有大小起伏（如 头部→主角区 用 `$spacing-xl(32)`，主角区内卡间 `$spacing-sm(12)`）。
- ❌ 全页统一 16px 间距、用分割线代替间距分组。

### 1.4 制造浮起（阴影层级）

**原理**：阴影代表层级。主角区浮起更高，次要区扁平。**不是每张卡都要阴影。**

**阴影档**（来自 `design-spec.md §1.7`）：`$shadow-sm / base / md / lg / xl`

**配方**：
- 首屏主角卡 → `$shadow-md`
- 普通内容卡 → `$shadow-base`
- 扁平信息条 / 列表内分组 → 无阴影，靠底色或间距区分
- 底部固定操作栏 → `$shadow-md`（向上投影）
- 弹窗 / 抽屉 → `$shadow-lg`

- ❌ 全页卡片同一档阴影、或处处重阴影。

### 1.5 色彩点睛（80% 灰白 + 局部品牌色）

**原理**：页面 **80%+ 面积是灰白**（`$color-bg-page(#F5F6FA)` 页底、`$color-bg-card(#FFFFFF)` 卡片），品牌色 `$color-primary(#0E74FF)` 只落在**按钮、状态标签、关键指标**等点睛位。

- ✅ 大面积留白 + 焦点处品牌色。
- ❌ 每张卡都有彩色背景 / 每个图标配不同色底色（彩虹图标）。
- 渐变只允许一级页面贯穿式头部（`linear-gradient(135deg, primary→primary-dark)`），二级页面禁渐变（见 `page-hierarchy.md`）。

### 1.6 状态表达（四层业务状态色）

**原理**：业务状态（待审/通过/驳回/草稿）必须用 `design-spec.md §1.3` 的**四层状态色体系**，不自造颜色。每个状态 = 主色 / 浅底 / 描边 / 文字 四件套。

| 语义 | 主色 | 浅底 | 文字 | 典型业务态 |
|------|------|------|------|-----------|
| 品牌/激活 | `$status-brand-main(#6D5CF6)` | `$status-brand-bg` | `$status-brand-text` | 进行中、已激活 |
| 成功 | `$status-success-main(#22C55E)` | `$status-success-bg` | `$status-success-text` | 已通过、已生效 |
| 警示 | `$status-warning-main(#F59E0B)` | `$status-warning-bg` | `$status-warning-text` | 待审批、待处理 |
| 错误 | `$status-error-main(#EF4444)` | `$status-error-bg` | `$status-error-text` | 已驳回、已逾期 |
| 信息 | `$status-info-main(#3B82F6)` | `$status-info-bg` | `$status-info-text` | 已抄送、草稿 |

StatusTag = 浅底 + 描边 + 文字色，配 `$radius-sm(4px)`~`$radius-md(8px)`。

## §2 布局原型库 —— 首屏骨架

8 种首屏结构原型，每种标注**视觉重心**和**节奏占比**。

**用法：3 个 UI 方案必须从本库选 3 个本质不同的原型**（信息架构差异），杜绝「换汤不换药」。ASCII 中 `█` 表示视觉重心区块，方块大小 ≈ 占屏比例。

### 原型 1 · 大卡主导
**定位**：首屏一个主角大卡占主要面积，用户最关心的内容直给。
```
┌──────────┐  头部（窄）
├──────────┤
│██████████│  ← 主角大卡（占屏 55-65%）：待办列表 / 核心内容
│██████████│
├──────────┤
│ ▒▒  ▒▒  ▒│  ← 次要信息条（扁平）
└──────────┘
```
**焦点**：主角卡内的首行/首项。**适合**：待办驱动的工作台、单一核心任务页。

### 原型 2 · 数据面板 + 入口
**定位**：顶部关键指标带，下方功能入口 + 列表。
```
┌──────────┐  头部
│ ██ ██ ██ │  ← 指标带（3 数字，占屏 25%）█焦点在第一个
├──────────┤
│ ▫ ▫ ▫ ▫  │  ← 金刚区入口
├──────────┤
│ ▒▒▒▒▒▒▒ │  ← 列表
└──────────┘
```
**焦点**：指标带首个数字。**适合**：管理者视角、综合工作台。

### 原型 3 · Feed 信息流
**定位**：纵向等权卡片流，无单一主角，靠卡片内层次。
```
│ ▒▒▒▒▒▒▒ │
│ ▒▒▒▒▒▒▒ │  ← 等宽卡片依次排列
│ ▒▒▒▒▒▒▒ │  （卡内自有标题/正文/状态层次）
```
**焦点**：每卡内部的标题 + 状态。**适合**：消息流、动态、通知中心。

### 原型 4 · 分段 Tab
**定位**：顶部分段切换，内容区随 Tab 变化。
```
│ [全部][待审][已通过] │  ← 分段器吸顶
├──────────┤
│ ▒▒▒▒▒▒▒ │  ← 当前段内容
│ ▒▒▒▒▒▒▒ │
```
**焦点**：选中段 + 内容首项。**适合**：多状态列表、需要分类浏览的场景。

### 原型 5 · 时间线 / 审批流
**定位**：纵向时间轴，强调流程顺序与节点状态。
```
│ ● 发起   张三 10:30 │
│ │                  │
│ ● 审批中 李四 ──── │  ← 当前节点高亮
│ ┊                  │
│ ○ 待签                │
```
**焦点**：当前进行节点。**适合**：审批流、物流跟踪、操作日志。

### 原型 6 · 列表密集型
**定位**：搜索筛选吸顶 + 高密度列表，强调检索效率。
```
│ 🔍 搜索        筛选▼ │ ← 吸顶
├──────────┤
│ ▒▒▒▒ ▒▒ │
│ ▒▒▒▒ ▒▒ │  ← 紧凑列表项
│ ▒▒▒▒ ▒▒ │
```
**焦点**：搜索框 + 每项状态/金额。**适合**：合同列表、审批记录等查找型页面。

### 原型 7 · 仪表盘
**定位**：图表为主角 + 指标卡环绕。
```
│ ██ ██ ██ │  指标卡
├──────────┤
│██████████│  ← 主图表（占屏 40%）
│██████████│
├──────────┤
│ ▒ TOP5   │
```
**焦点**：主图表 + 核心指标。**适合**：数据看板、业绩驾驶舱。

### 原型 8 · 表单分组
**定位**：分组卡片纵向排列 + 底部固定提交。
```
│ ┌ 基本信息 ┐ │
│ │ ▫▫▫▫▫▫ │ │  ← 分组卡
│ └────────┘ │
│ ┌ 签署信息 ┐ │
│ │ ▫▫▫▫   │ │
│ └────────┘ │
├──────────┤
│ [  提交  ] │ ← 底部固定
```
**焦点**：当前填写字段 + 提交按钮。**适合**：新建/编辑类表单。

## §3 风格人格库 —— 视觉气质基调

4 种人格，每种是一套打包好的视觉配方（凌驾于 主题预设 / Tabbar 风格 / 页面层级 之上的「气质层」）。**AI 按场景自动推荐一种**（见 `design-guidance.md` 匹配矩阵），告知用户、用户可改、写入 `.mug-project.json` 的 `personality` 锁定。

### 人格 1 · 克制专业（Restrained Professional）
> 性格：像资深行政/法务，秩序感强、不花哨、信息高效。

| 维度 | 配方 |
|------|------|
| 主题 | `blue.json`（商务蓝） |
| 圆角 | 卡片 `$radius-md(8px)`，一级页面面板 `$radius-xl(16px)` |
| 阴影 | `$shadow-base` 为主，主角卡 `$shadow-md` |
| 装饰 | 极简，仅一级头部几何圆（`rgba(255,255,255,0.08)`） |
| 色彩 | 品牌蓝 + 中性灰，80%+ 灰白，禁饱和彩底 |
| 字重对比 | `$font-weight-medium(500)` ↔ `$font-weight-bold(600)` |
| 搭配 Tabbar | 风格 A 沉浸式头部 / 风格 C 数据看板 |

**适合**：OA、审批、用印、合同、政企内部应用。**反面**：别做成彩色金刚区 + 插画的 C 端感。

### 人格 2 · 精致现代（Refined Modern）
> 性格：像高端 SaaS（Linear/Notion 气质），靠留白和排版打磨高级感。

| 维度 | 配方 |
|------|------|
| 主题 | `blue.json`（偏冷静） |
| 圆角 | `$radius-lg(12px)` 统一 |
| 阴影 | `$shadow-sm`/`$shadow-base`，强调微妙 |
| 装饰 | 几乎无，靠大留白（区块间 `$spacing-xl(32)`+） |
| 色彩 | 高级灰为底，品牌色克制点睛 |
| 字重对比 | 字号跨度拉大（`$font-size-base(14)` ↔ `$font-size-xl(20)`） |
| 搭配 Tabbar | 风格 B 卡片金刚区（精修版） |

**适合**：工具型 SaaS、智能办公、面向专业用户的产品。**反面**：别堆装饰、别用重阴影。

### 人格 3 · 友好活力（Friendly Energetic）
> 性格：像 C 端消费 App，温暖、亲和、有插画。

| 维度 | 配方 |
|------|------|
| 主题 | `orange.json` / `green.json` |
| 圆角 | `$radius-xl(16px)` / `$radius-xxl(18px)` |
| 阴影 | `$shadow-md`，更显浮起 |
| 装饰 | 允许浅色块底、品牌插图（SVG）、点状纹理 |
| 色彩 | 暖色渐变 + 温暖辅助色，允许局部彩底 |
| 字重对比 | `$font-weight-bold(600)` 标题 + 圆润正文 |
| 搭配 Tabbar | 风格 D 图文并茂 |

**适合**：C 端消费、教育、健康、生活服务。**反面**：别做成冷冰冰的表格后台。

### 人格 4 · 数据驱动（Data-Centric）
> 性格：像 BI 驾驶舱，图表说话、信息密度高、对比强。

| 维度 | 配方 |
|------|------|
| 主题 | `blue.json` |
| 圆角 | `$radius-md(8px)`（紧凑） |
| 阴影 | `$shadow-base` |
| 装饰 | 仅几何辅助线，禁纹理（避免干扰读数） |
| 色彩 | 品牌蓝 + 功能色（涨 `$color-success` / 跌 `$color-danger`）高对比 |
| 字重对比 | 指标数值 `$font-size-hero(26)+$font-weight-heavy(700)` |
| 搭配 Tabbar | 风格 C 数据看板 + 原型 7 仪表盘 |

**适合**：管理看板、统计报表、数据中台。**反面**：别用大圆角大留白稀释信息密度。

## §4 需求扩展法 —— 一句话 → C 级 prompt 四步

拿到「合同审批工作台」这种一句话需求，按以下四步展开：

**第 1 步 · 定人格基调** —— 查 `design-guidance.md` 匹配矩阵（行业 + 人群 → 人格）。
> 「合同审批」= B 端政企 → **克制专业** → 自动带出 blue 主题 + Tabbar 风格 A + 极简装饰。读 `.mug-project.json`，若已有 `personality` 则沿用。

**第 2 步 · 选布局原型，定首屏重心** —— 从 §2 选一个原型，明确谁占屏主要面积。
> 审批人最关心待办 → **原型 1 大卡主导**，待办列表大卡占屏 ~60%。

**第 3 步 · 逐区块填视觉指令** —— 对每个区块，依次回答展开追问清单（见 `pencil-prompt-template.md`）：尺寸 / 内距？背景（底色 + 圆角档 + 阴影档）？内部元素字号档 + 字重档 + 色彩 Token？焦点是谁？与上下区块间距节奏？全程 §1 配方 + Token 双标注。

**第 4 步 · 全局节奏校验** —— 对照 `prompt-rules.md` 的 **C 级自检清单**逐条核对，缺哪条补哪条。

> 完整走查产物见 `visual-prompt-examples/workbench.md`（工作台）、`form.md` / `list.md` / `detail.md`。
