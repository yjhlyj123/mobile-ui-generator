# 页面层级设计差异体系

> 一级页面（Tabbar）与二级页面在视觉表现上必须形成明显的层次差异，让用户感知到不同层级的信息架构。

---

## 1. 定义

| 层级 | 定义 | 典型页面 |
|------|------|----------|
| **一级页面** | Tabbar 直达的核心入口页 | 首页、工作台、消息、我的 |
| **二级页面** | 从一级页面进入的功能页 | 列表、详情、表单、搜索结果 |
| **三级页面** | 从二级页面进入的深层页 | 编辑、确认、设置子项 |

---

## 2. 视觉差异对照表

| 维度 | 一级页面 | 二级页面 |
|------|----------|----------|
| **导航栏** | 自定义 / 沉浸式 / 隐藏 | 标准导航栏 + 返回按钮 |
| **头部区域** | 品牌化贯穿式头部（见 § 3.1）| 纯白或透明 |
| **圆角** | 大圆角 `$radius-xxl`（18px+） | 标准圆角 `$radius-md`（8px） |
| **首屏信息** | 问候语、统计卡片、金刚区 | 直接展示业务数据 |
| **装饰元素** | 允许品牌化几何装饰 / 微弱纹理 | 禁止装饰性元素 |
| **标题字号** | 大标题 `$font-size-hero` | 标准 `$font-size-lg` |
| **标题字重** | `$font-weight-black`（800） | `$font-weight-bold`（600） |
| **阴影** | 允许 `$shadow-md` 以上 | 仅 `$shadow-base` |
| **Tabbar** | 可见、带选中态动效 | 隐藏 / 不可见 |

---

## 3. 一级页面设计规范

### 3.0 微信小程序适配

一级页面使用自定义导航栏时，**必须**适配微信小程序的胶囊按钮区域。

#### 胶囊区域约束

```
┌──────────────────────────────────────────────┐
│                状态栏                          │ ← statusBarHeight
├────────────────────────────────┬─────────────┤
│                                │ ╭─────────╮ │
│  标题 / 问候语 / 搜索框        │ │  胶囊按钮 │ │ ← 不可覆盖区域
│                                │ ╰─────────╯ │
├────────────────────────────────┴─────────────┤
│                                               │
│          贯穿式头部背景继续延伸                  │ ← 背景不截断
│                                               │
│  ┌───────────────────────────────────────┐    │
│  │         金刚区 / 统计卡片              │    │ ← 悬浮在渐变区域上
│  └───────────────────────────────────────┘    │
│                                               │
├───────────────────────────────────────────────┤
│          内容区域（白底卡片）                    │
└───────────────────────────────────────────────┘
```

#### 安全高度计算

```typescript
const menuButton = wx.getMenuButtonBoundingClientRect()
const { statusBarHeight } = wx.getSystemInfoSync()

// 自定义导航栏高度
const navBarHeight = (menuButton.top - statusBarHeight) * 2 + menuButton.height
// 总高度 = 状态栏 + 导航栏
const totalNavHeight = statusBarHeight + navBarHeight

// 胶囊右侧安全间距
const capsuleRight = menuButton.right + 12 // 胶囊右边 + 12px 安全区
```

#### 适配要点

| 规则 | 说明 |
|------|------|
| **右侧安全区** | 导航栏内容右边距 ≥ `胶囊宽度 + 12px`，避免内容被胶囊遮挡 |
| **高度对齐** | 导航栏标题/图标垂直居中于胶囊区域 |
| **贯穿式背景** | 头部渐变/装饰从状态栏顶部延伸到内容区，导航栏叠加其上 |
| **H5 降级** | H5 无胶囊，导航栏高度固定 `44px`，顶部偏移 `var(--status-bar-height, 0px)` |

### 3.1 头部区域

一级页面头部是品牌差异化的核心区域，**必须**使用**贯穿式头部**。

#### 贯穿式头部定义

头部背景从状态栏顶部一直延伸到内容区上方，自定义导航栏叠加在背景之上，背景不被任何元素截断。渐变区域通常占首屏 30%-40%。

#### 背景设计元素

一级页面头部**禁止使用纯色背景**，必须包含品牌化设计元素。提供两种预设风格：

**风格 A — 渐变 + 几何装饰**

品牌渐变底色 + 右下角低透明度圆形 / 弧线装饰。克制、专业。

```scss
.header-geo {
  background: linear-gradient(135deg, var(--mug-color-primary), var(--mug-color-primary-dark));
  position: relative;
  overflow: hidden;

  &::before {
    content: '';
    position: absolute;
    right: -40rpx;
    bottom: -40rpx;
    width: 240rpx;
    height: 240rpx;
    border-radius: 50%;
    background: rgba(255, 255, 255, 0.08);
  }

  &::after {
    content: '';
    position: absolute;
    right: 80rpx;
    bottom: 40rpx;
    width: 120rpx;
    height: 120rpx;
    border-radius: 50%;
    background: rgba(255, 255, 255, 0.05);
  }
}
```

**风格 B — 渐变 + 微弱纹理**

品牌渐变底色 + 点状 / 网格 / 斜线微纹理叠加。更有质感。

```scss
.header-texture {
  background: linear-gradient(135deg, var(--mug-color-primary), var(--mug-color-primary-dark));
  position: relative;

  &::before {
    content: '';
    position: absolute;
    inset: 0;
    // 点状纹理（1px 间隔 12px）
    background-image: radial-gradient(circle, rgba(255,255,255,0.06) 1px, transparent 1px);
    background-size: 24rpx 24rpx;
  }
}
```

**Agent 选择规则**：

| 场景 | 推荐风格 |
|------|----------|
| B 端商务 / OA / 审批 / 金融 | 风格 A（几何装饰）— 简洁专业 |
| B 端数据管理 / 看板 | 风格 A — 避免纹理干扰数据阅读 |
| C 端消费 / 教育 / 健康 | 风格 B（纹理）— 质感更丰富 |
| 用户明确要求"简约" | 风格 A |
| 用户明确要求"有质感" | 风格 B |

**禁止**：
- 大面积图片背景
- 营销 Banner
- 花哨粒子效果 / Lottie 动效
- 纯色（无渐变、无装饰元素）

头部内容可包含：
- 问候语 + 用户名（如："你好，张三"）
- 搜索框（带圆角和透明背景）
- 通知/消息入口图标（注意避开胶囊区域）

### 3.2 金刚区（Quick Actions）

金刚区是一级页面的功能导航核心：

```
┌─────────────────────────────────────────┐
│  ┌──────┐  ┌──────┐  ┌──────┐  ┌──────┐│
│  │ Icon │  │ Icon │  │ Icon │  │ Icon ││
│  │ 文字 │  │ 文字 │  │ 文字 │  │ 文字 ││
│  └──────┘  └──────┘  └──────┘  └──────┘│
│  ┌──────┐  ┌──────┐  ┌──────┐  ┌──────┐│
│  │ Icon │  │ Icon │  │ Icon │  │ Icon ││
│  │ 文字 │  │ 文字 │  │ 文字 │  │ 文字 ││
│  └──────┘  └──────┘  └──────┘  └──────┘│
└─────────────────────────────────────────┘
```

- 每行 4-5 个入口
- 图标使用主题色渐变或纯色圆形底色
- 文字使用 `$font-size-sm`

### 3.3 统计/概览卡片

一级页面可使用大面积统计卡片展示核心数据：

```
┌──────────────────┐  ┌──────────────────┐
│      待审批       │  │      已完成       │
│       12         │  │       86         │
│   本周 +3        │  │   本月 +24       │
└──────────────────┘  └──────────────────┘
```

- 数字使用 `$font-size-xl` + `$font-weight-heavy`
- 卡片可带轻微渐变底色或品牌色点缀
- 允许 `$shadow-md`

### 3.4 内容分区

一级页面的内容区域按模块分区，每个分区包含：
- 区块标题 + "查看更多"入口
- 内容列表/卡片
- 区块间使用 `$spacing-lg`（24px）间距

## 4. 二级页面设计规范

### 4.1 导航栏

- 使用标准导航栏组件
- 左侧返回按钮
- 中部标题
- 右侧操作按钮（可选）

### 4.2 内容布局

- 统一使用标准卡片布局
- 圆角 `$radius-md`（8px）
- 无装饰性元素
- 阴影仅使用 `$shadow-base`

### 4.3 操作区域

- 底部固定操作栏使用 `$shadow-md` 向上投影 + `safe-area-bottom`
- 按钮遵循标准尺寸
- 按钮布局规范详见 `design-spec.md` § 7.3（详情页）和 § 5.4（表单）

---

## 5. 风格选择流程

当 Agent 检测到页面为 Tabbar 页面（一级页面）时：

1. **必须**提供至少 3 种差异化的一级页面风格方案
2. 每种方案需说明：头部处理、金刚区样式、内容分区策略
3. 每种方案的头部必须注明选用的背景风格（A 几何装饰 / B 微弱纹理）
4. 等待用户选择后再进入实现
5. 详细的风格预设见 `core/tabbar-styles.md`

## 6. 自检清单

- [ ] 一级页面是否使用了贯穿式品牌化头部（非纯色）？
- [ ] 一级页面头部是否适配了微信胶囊区域？
- [ ] 一级页面的圆角是否 ≥ `$radius-xxl`？
- [ ] 二级页面是否保持了朴素的标准布局？
- [ ] 一级页面与二级页面的视觉差异是否明显？
- [ ] 一级页面是否过度装饰（是否仍保持信息克制）？
- [ ] 二级页面是否误用了一级页面的渐变/插图/大标题？
