# UI 方案标准化输出模板

> 进入复杂模式（`/mug:x`）、设计先行（`/mug:d`）或 Tabbar 页面时，Agent 必须用本模板输出**至少 3 个差异化方案**。用户选定前，不得进入结构化 Prompt 或代码实现。
>
> **配合使用**：`design-arsenal.md §2`（布局原型库）+ `§3`（风格人格库）。方案差异的根本来源就是「选 3 个本质不同的布局原型」。

---

## 方案差异化的来源（铁律）

生成方案前，Agent 先内部推导（不输出，但必须体现在方案中）：
1. **用户核心任务**：用户打开这个页面最想完成什么？
2. **信息优先级**：哪些必须首屏可见，哪些可下沉？
3. **操作频率**：哪些高频、哪些低频？

然后据此，**从 `design-arsenal.md §2` 布局原型库选 3 个本质不同的原型**，每个原型代表一种不同的设计假设：

- 例：方案 A = 原型 1 大卡主导（假设用户最关心待办，待办占屏 60%）
- 方案 B = 原型 2 数据面板+入口（假设用户最关心全局数据，指标带打头）
- 方案 C = 原型 5 时间线（假设用户最关心流程进度，时间轴纵向浏览）

**强制约束：**
- 三个方案的**布局原型必须不同** → 首屏重心不同 → 用户动线不同（这是信息架构差异，不是换皮）。
- **禁止**三个方案都是「渐变头部 + 金刚区 + 列表」的同一模板变体。
- 每个方案标注：**采用的布局原型 + 风格人格基调 + 首屏视觉重心**。
- ASCII 布局图必须用 `█`（视觉重心）/ `▒`（次要）体现**区块大小比例**（视觉节奏），并标注真实业务内容（如「待审批 3 件」），不用「模块1」。
- **禁止空洞形容词**：不用「简约大气」「现代感」「高端商务」，用具体布局和内容体现风格。

## 单方案模板

```md
## 方案 [序号]：[方案名称]

- **布局原型**：[design-arsenal §2 中的原型名，如「原型 1 大卡主导」]
- **人格基调**：[design-arsenal §3 中的人格，如「克制专业」]（出方案时告知用户，用户可改）
- **首屏视觉重心**：[哪个区块占屏主要面积]

### 一句话定位
这个方案假设用户最想 ___，因此把 ___ 放在首屏主角位。
（示例：「假设用户最关心待处理事项，首屏用大卡列出待办，占屏 60%，快速进入处理。」）

### 用户动线
1. 打开 → 首先看到 ___
2. 扫一眼 → 了解到 ___
3. 点击 → 进入 ___

### 布局示意（ASCII，█ 重心 / ▒ 次要，方块大小≈占屏比，标真实业务内容）

### 首屏信息分配
| 区域 | 承载内容 | 占屏比 | 为什么放这里 |
|------|----------|--------|--------------|
| ___ | ___ | ___% | ___ |

### 视觉节奏（区别于其他方案的关键）
- 主角区：___（大）｜ 次要区：___（中/小）｜ 间距节奏：___

### 适合 / 不适合
- 适合：用户主要是 ___、最高频操作是 ___、最重要信息是 ___
- 不适合：如果 ___
```

## 方案对比总结模板

```md
## 方案对比总结

| 维度 | 方案 A | 方案 B | 方案 C |
|------|--------|--------|--------|
| 布局原型 | ___ | ___ | ___ |
| 人格基调 | ___ | ___ | ___ |
| 核心假设 | 用户最想 ___ | ___ | ___ |
| 首屏主角（占屏） | ___（__%） | ___ | ___ |
| 用户动线 | 先看→再做 | 先做→再看 | 边看边做 |
| 信息密度 | 高/中/低 | ___ | ___ |

### 选择建议
- **选方案 A**：如果 ___（具体业务场景）
- **选方案 B**：如果 ___
- **选方案 C**：如果 ___

> 请选择一个方案编号（A/B/C），或描述偏好方向。也可混搭：「用 A 的头部 + B 的内容区」。
> 也可以换人格基调（如「我想要更活力一点」）。
> 选定后我会先用 Pencil 生成视觉设计稿给你预览，你确认满意后再进入代码实现。
```

## Tabbar 页面专用扩展

当方案涉及 Tabbar 一级页面时，在单方案模板中补充（参照 `tabbar-styles.md` 的 A/B/C/D 风格）：

```md
- **Tabbar 风格**：[A 沉浸式头部 / B 卡片金刚区 / C 数据看板 / D 图文并茂]
- **头部背景**：渐变 + 几何装饰（克制专业/数据驱动）/ 渐变 + 纹理（友好活力）
- **金刚区**：有 / 无 / 可滚动（按实际入口数量决定，非必须）
```

## 使用规则

1. **至少 3 个方案**，且**布局原型必须各不相同**（信息架构差异优先于视觉差异）。
2. **每个方案标注布局原型 + 人格基调**，ASCII 图体现占屏比例与真实业务数据。
3. **给出选择建议**：对比总结必须含「什么情况选什么」的具体指引，不只说「请选择」。
4. **明确等待**：输出方案后必须停下等用户选择，禁止自动进入 Pencil prompt 或代码。
5. **可迭代**：不满意则提供新方案（可换布局原型或人格）或局部微调。
6. **禁止 AI 套路**：见 `design-arsenal.md §3` 各人格的「反面」与 `design-spec.md §9`。
