# 移动端 UI 设计规范 v5.0

> 适用范围：微信小程序 + H5 | 技术栈：uni-app + Vue 3 + UnoCSS + SCSS
> 组件库：wot-design-uni (小程序+H5, 前缀 `wd-`) / Vant 4 (仅H5, 前缀 `van-`)
> 主题配置：`theme.json`（单源真理） | Token 文件：`core/tokens/`

---

## 0. 设计哲学

> **核心理念：克制而不传统，新颖而不花里胡哨。**

| 原则 | 说明 | 正例 | 反例 |
|------|------|------|------|
| **信息克制** | 每个区块只传达一个核心信息，拒绝堆砌 | 统计卡片只展示数字 + 趋势 | 卡片内同时放数字、图表、说明文字、操作按钮 |
| **留白即设计** | 用间距而非装饰表达层次 | 模块间 `$spacing-lg`（24px） | 模块间用分割线 + 色块 + 图标分隔 |
| **功能驱动装饰** | 装饰元素服务于信息架构，仅一级页面头部允许品牌化元素 | 头部渐变 + 低透明度几何装饰引导视觉焦点 | 二级页面也加渐变头部和插图 |
| **微交互代替大动效** | `$duration-fast`（150ms）的按钮反馈 > 花哨的 Lottie | 按钮按压态 0.95 缩放 + 颜色变深 | 页面加载播放 2 秒品牌动画 |
| **一致性 > 个性化** | 所有页面共享同一套 Token，不允许页面级特殊处理 | 所有卡片统一 `$radius-md` | "这个页面特殊一点，用 12px 圆角" |
| **组件优先** | 优先使用 `wot-design-uni` 标准组件，仅在组件无法满足时自定义 | `wd-cell-group` 实现表单 | 手写 div 模拟表单布局 |
| **真实感优先** | 设计应像有经验的人类设计师手工打磨，拒绝 AI 批量生成的模板感 | 首屏有明确视觉重心，区块有大小主次之分，间距有节奏变化 | 所有卡片等大等高、2×2 网格完美对称、渐变头部 + 圆形图标金刚区的万能模板 |

## 1. Design Tokens

Token 体系分三层：基础层 → 语义层 → 主题层。

- **基础层**：`core/tokens/base-tokens.scss` — 原始色值、字号、间距等
- **语义层**：`core/tokens/semantic-tokens.scss` — CSS Custom Properties 映射
- **主题层**：`core/tokens/theme-presets/*.json` — 可切换的主题配置
- **类型契约**：`core/tokens/theme-contract.ts` — TypeScript 类型定义

## 2. 布局规范

### 2.1 页面容器
- 页面底色固定为 `$color-bg-page` (`#F5F6FA`)。
- 内容区域使用 `safe-area` (`px-16px`)。

### 2.2 卡片布局
- 背景：`#FFFFFF`。
- 圆角：标准页 `8px`，"我的"页 `18px`。
- 边框：**默认无边框**。
- 阴影：`0 2px 8px rgba(0,0,0,0.06)`（仅用于 `card-shadow`）。

### 2.3 筛选器 (Filter Pill)
- 激活态：`bg-#2563EB text-white`。
- 非激活态：`bg-#F3F4F6 text-#475569`。

### 2.4 微信小程序平台约束

微信小程序的胶囊按钮（右上角）会占用导航栏区域，一级页面使用自定义导航栏时**必须**适配。

#### 胶囊区布局示意

```
┌─────────────────────────────────────────┐
│ 状态栏 (statusBarHeight)                 │
├──────────────────────────────┬──────────┤
│                              │ ┌──────┐ │
│   自定义导航内容              │ │ 胶囊  │ │
│   (标题/搜索/问候语)          │ │ 按钮  │ │
│                              │ └──────┘ │
├──────────────────────────────┴──────────┤
│ 页面内容区域                              │
└─────────────────────────────────────────┘
```

#### 安全高度计算

```typescript
// 获取胶囊按钮位置
const menuButton = wx.getMenuButtonBoundingClientRect()
// 状态栏高度
const { statusBarHeight } = wx.getSystemInfoSync()

// 自定义导航栏高度（推荐固定计算方式）
const navBarHeight = (menuButton.top - statusBarHeight) * 2 + menuButton.height
// 导航栏总高度 = 状态栏 + 导航栏
const totalNavHeight = statusBarHeight + navBarHeight
```

#### 适配规则

| 规则 | 说明 |
|------|------|
| **右侧安全区** | 导航栏内容右边距 ≥ 胶囊宽度 + `12px` |
| **高度对齐** | 自定义导航栏标题/图标垂直居中于胶囊区域 |
| **贯穿式头部** | 一级页面头部背景从状态栏顶部延伸到内容区，自定义导航栏叠加其上，背景不被截断 |
| **H5 降级** | H5 环境无胶囊，导航栏高度固定 `44px`，顶部取 `var(--status-bar-height, 0px)` |

#### UnoCSS Shortcut 建议

```typescript
// 已有的 safe-area 处理底部
'safe-area': 'px-16px pb-[env(safe-area-inset-bottom)]'
// 建议新增：导航栏安全右间距
'nav-safe-right': 'pr-[calc(var(--mug-capsule-width,87px)+12px)]'
```

## 5. 表单设计规范

### 5.1 表单布局

使用 `wd-cell-group` 作为表单容器，`wd-cell` 作为字段行。

```
┌─────────────────────────────────────────┐
│ 基本信息                          [必填] │  ← PageSection 标题
├─────────────────────────────────────────┤
│ * 合同名称              请输入合同名称   │  ← wd-cell + wd-input
│─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─│  ← hairline 分割线
│ * 合同类型                      请选择 ▸│  ← wd-cell + wd-picker
│─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─│
│   备注说明              请输入备注信息   │  ← 非必填，无星号
├─────────────────────────────────────────┤
│                                         │
│ 签署信息                                 │  ← 第二个 PageSection
├─────────────────────────────────────────┤
│ * 签署日期              请选择日期     ▸│
│─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─│
│ * 签署方                请选择签署方   ▸│
└─────────────────────────────────────────┘
         ...（更多字段）...
┌─────────────────────────────────────────┐
│          [ 保存草稿 ]    [ 提 交 ]       │  ← 底部固定操作栏
└─────────────────────────────────────────┘
```

### 5.2 字段规范

| 维度 | 规范 |
|------|------|
| **字段高度** | 正常模式 `88rpx`，紧凑模式 `72rpx` |
| **标签（label）** | **左对齐**（`text-align: left`），`$font-size-base`，`$color-text-regular`，宽度建议 `180rpx` |
| **输入区** | **右对齐**（`text-align: right`），`$font-size-base`，`$color-text-primary` |
| **占位符** | **右对齐**（与输入区对齐），`$color-text-placeholder` |
| **必填标记** | label 前红色星号 `*`，使用 `$color-danger` |
| **分割线** | 组内使用 `hairline` 分割线（`$color-border-light`），组间使用 `$spacing-lg` 留白 |
| **分组** | 相关字段用 `PageSection` 包裹，组标题用 `$font-size-md` + `$font-weight-medium` |

### 5.3 校验规则

| 维度 | 规范 |
|------|------|
| **触发时机** | 字段失焦（blur）时校验，提交时全量校验 |
| **错误提示** | 字段下方显示，**右对齐**（与输入值对齐），`$color-danger` + `$font-size-sm`（12px） |
| **错误态** | 字段右侧或底部边框变为 `$color-danger` |
| **成功态** | 不做视觉反馈（克制原则，减少干扰） |
| **提交按钮** | 未通过校验时使用 `$color-primary-disabled`，禁止点击 |

### 5.4 表单类型适配

| 字段类型 | 推荐组件 | 说明 |
|----------|----------|------|
| 文本输入 | `wd-input` | 单行文本 |
| 多行文本 | `wd-textarea` | 自动增高，最大 `200rpx` |
| 选择器 | `wd-picker` / `wd-select-picker` | 点击后弹出选择面板 |
| 日期选择 | `wd-datetime-picker` | 年-月-日 / 年-月-日 时:分 |
| 开关 | `wd-switch` | 布尔值切换 |
| 上传 | `wd-upload` | 图片/文件上传，带预览 |
| 评分 | `wd-rate` | 星级评分 |

### 5.5 移动端/小程序表单对齐统一规范

> 适用范围：微信小程序 + H5，所有表单页面强制遵循。

#### 对齐总则

移动端表单采用 **「左 label — 右 value」** 的经典双栏对齐布局，确保视觉一致性和可读性。

```
┌─────────────────────────────────────────┐
│ * 合同名称              采购服务合同-年度│  ← label 左对齐 | value 右对齐
│─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─│
│ * 合同类型                      请选择 ▸│  ← label 左对齐 | placeholder 右对齐
│─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─│
│ * 合同金额                              │
│                        请输入合同金额    │  ← placeholder 右对齐
│                   金额必须大于0          │  ← 校验提示右对齐，与值对齐
└─────────────────────────────────────────┘
```

#### 对齐规则明细

| 元素 | 对齐方式 | 说明 |
|------|----------|------|
| **label（标签）** | `text-align: left` | 始终靠左，包含必填星号 `*` 时星号在 label 最前方 |
| **value（表单值）** | `text-align: right` | 输入值、已选值、显示值统一靠右 |
| **placeholder** | `text-align: right` | 与 value 位置一致，用户输入后无缝替换 |
| **校验错误提示** | `text-align: right` | 显示在字段下方，与 value 区域右对齐 |
| **必填星号** | 跟随 label 左对齐 | 位于 label 文字最前方，使用 `$color-danger` |

#### CSS 实现参考

```scss
// 表单对齐核心样式
.form-field {
  // label 左对齐
  :deep(.wd-cell__title) {
    text-align: left;
  }

  // value、placeholder 右对齐
  :deep(.wd-cell__value),
  :deep(.wd-input__inner),
  :deep(.wd-textarea__inner) {
    text-align: right;
  }

  // placeholder 右对齐（通过 input 的属性传递）
  :deep(.wd-input__inner)::placeholder,
  :deep(.wd-textarea__inner)::placeholder {
    text-align: right;
  }

  // 校验错误提示右对齐
  .form-error-msg {
    text-align: right;
    color: $color-danger;
    font-size: $font-size-sm;
    padding-right: $spacing-base;
    margin-top: $spacing-xxs;
  }
}
```

#### 特殊字段对齐处理

| 字段类型 | 对齐说明 |
|----------|----------|
| **单行文本** (`wd-input`) | value + placeholder 右对齐 |
| **多行文本** (`wd-textarea`) | 独占一行时左对齐（因内容较长），内嵌 cell 时右对齐 |
| **选择器** (`wd-picker`) | 已选值右对齐，placeholder 右对齐，右箭头 `▸` 紧跟其后 |
| **日期选择** (`wd-datetime-picker`) | 同选择器规则 |
| **开关** (`wd-switch`) | 组件本身靠右放置 |
| **上传** (`wd-upload`) | 独占区域，左对齐展示（因需要预览空间） |
| **评分** (`wd-rate`) | 组件靠右放置 |

#### 校验提示对齐示意

```
┌─────────────────────────────────────────┐
│ * 手机号码                  1381234567  │  ← 正常态：value 右对齐
│─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─│
│ * 邮箱地址                  abc@invalid │  ← 错误态：value 右对齐
│                     请输入正确的邮箱格式 │  ← 错误提示：右对齐，与 value 对齐
│─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─│
│ * 合同金额                              │  ← 空值错误态
│                        请输入合同金额    │  ← 错误提示：右对齐
└─────────────────────────────────────────┘
```

#### 强制约束（Agent 必须遵守）

1. **不允许 label 右对齐或居中**：label 始终 `text-align: left`
2. **不允许 value 左对齐**：表单值始终 `text-align: right`（多行文本独占行除外）
3. **placeholder 必须与 value 同侧**：确保用户输入前后视觉位置一致
4. **校验提示必须与 value 对齐**：错误信息出现在用户视线焦点附近，便于定位问题字段
5. **跨端一致**：微信小程序和 H5 端必须表现一致，不允许端差异导致对齐不同

### 5.6 表单校验规则库

> 所有表单校验统一使用 `wd-form` 的 `rules` 属性配置，不手写校验逻辑。

#### 必填校验模板

```typescript
// 输入类字段
{ required: true, message: '请填写${label}' }
// 选择类字段（picker / datetime-picker / select-picker）
{ required: true, message: '请选择${label}' }
// 上传类字段
{ required: true, message: '请上传${label}' }
```

#### 格式校验规则表

| 规则 ID | 字段类型 | 正则表达式 | 错误提示 | 键盘类型 |
|---------|---------|-----------|---------|---------|
| `phone` | 手机号 | `/^1[3-9]\d{9}$/` | 请输入正确的手机号码 | `type="number"` |
| `tel` | 固定电话 | `/^0\d{2,3}-?\d{7,8}$/` | 请输入正确的固定电话 | `type="number"` |
| `idCard` | 身份证号 | `/^\d{17}[\dXx]$/` | 请输入正确的身份证号码 | `type="idcard"` |
| `email` | 邮箱 | `/^[^\s@]+@[^\s@]+\.[^\s@]+$/` | 请输入正确的邮箱地址 | `type="text"` |
| `amount` | 金额（≤2位小数） | `/^(0\|[1-9]\d*)(\.\d{1,2})?$/` | 请输入正确的金额（最多两位小数） | `type="digit"` |
| `integer` | 正整数 | `/^[1-9]\d*$/` | 请输入正整数 | `type="number"` |
| `nonNegInt` | 非负整数 | `/^\d+$/` | 请输入 0 或正整数 | `type="number"` |
| `percent` | 百分比（0-100） | `/^(100\|[1-9]?\d)(\.\d{1,2})?$/` | 请输入 0-100 之间的数值 | `type="digit"` |
| `creditCode` | 统一社会信用代码 | `/^[0-9A-HJ-NP-RT-UW-Y]{18}$/` | 请输入正确的 18 位社会信用代码 | `type="text"` |
| `bankCard` | 银行卡号 | `/^\d{16,19}$/` | 请输入正确的银行卡号 | `type="number"` |
| `zipCode` | 邮政编码 | `/^\d{6}$/` | 请输入正确的 6 位邮政编码 | `type="number"` |
| `url` | 网址 | `/^https?:\/\/.+/` | 请输入正确的网址（以 http:// 或 https:// 开头） | `type="text"` |
| `plateNo` | 车牌号 | `/^[京津沪渝冀豫云辽黑湘皖鲁新苏浙赣鄂桂甘晋蒙陕吉闽贵粤川青藏琼宁夏][A-Z][A-Z0-9]{4}[A-Z0-9挂学警港澳]$/` | 请输入正确的车牌号 | `type="text"` |
| `password` | 密码 | `/^(?=.*[A-Za-z])(?=.*\d).{8,20}$/` | 密码需 8-20 位，包含字母和数字 | `type="text"` |

```typescript
// 手机号字段 — 必填 + 格式校验

const amountRules = [

]

// 身份证字段 — 必填 + 格式校验
const idCardRules = [
  { required: true, message: '请填写身份证号码' },
  { pattern: /^\d{17}[\dXx]$/, message: '请输入正确的身份证号码' }

```

### 5.7 字段名智能推断映射表

> Agent 在生成表单代码时，必须根据字段名关键词自动匹配校验规则和键盘类型，无需用户逐一指定。

#### 推断映射表

| 字段名关键词 | 推断规则 ID | 键盘类型 | 示例字段名 |
|-------------|-----------|---------|-----------|
| `phone` / `mobile` / `手机` / `电话` / `联系方式` | `phone` | `number` | contactPhone、手机号码、联系电话 |
| `tel` / `座机` / `固话` / `固定电话` | `tel` | `number` | officeTel、公司座机 |
| `idCard` / `身份证` / `证件号` / `证件` | `idCard` | `idcard` | idCardNo、身份证号码 |
| `email` / `邮箱` / `mail` | `email` | `text` | userEmail、电子邮箱 |
| `amount` / `金额` / `price` / `价格` / `费用` / `cost` / `money` | `amount` | `digit` | totalAmount、合同金额、单价 |
| `count` / `数量` / `num` / `qty` / `件数` | `nonNegInt` | `number` | itemCount、采购数量 |
| `age` / `年龄` | `nonNegInt` | `number` | userAge、年龄 |
| `creditCode` / `信用代码` / `营业执照` / `社会信用` | `creditCode` | `text` | socialCreditCode、统一社会信用代码 |
| `bankCard` / `银行卡` / `卡号` / `账号`（银行语境） | `bankCard` | `number` | bankCardNo、银行卡号 |
| `zipCode` / `邮编` / `邮政编码` | `zipCode` | `number` | postCode、邮编 |
| `url` / `网址` / `链接` / `link` / `website` | `url` | `text` | websiteUrl、公司官网 |
| `plateNo` / `车牌` / `车牌号` | `plateNo` | `text` | carPlateNo、车牌号码 |
| `password` / `密码` / `pwd` | `password` | `text` | loginPassword、登录密码 |
| `percent` / `百分比` / `比例` / `rate`（非评分语境） | `percent` | `digit` | taxRate、折扣比例 |

#### 推断规则（Agent 强制遵守）

1. **先精确后模糊**：先按字段名精确匹配（如 `phone`），再按关键词包含匹配（如字段名含"电话"）
2. **非必填也校验格式**：匹配到格式规则后，即使字段非必填，用户有输入时也必须校验格式
3. **未匹配仅必填**：未匹配到任何格式规则的字段，仅在必填时添加 `required` 校验
4. **语境推断**：AI 可根据业务语境做合理推断（如"联系人电话" → `phone`，"开户行账号" → `bankCard`）
5. **冲突时取严格**：一个字段名匹配到多个规则时，取校验更严格的规则
6. **键盘类型必须同步**：推断出校验规则后，必须同时设置对应的 `type` 属性触发正确的键盘

### 5.8 字段联动规范

#### 联动类型

| 类型 | 说明 | 示例 |
|------|------|------|
| **显示/隐藏联动** | 字段 A 的值决定字段 B 是否显示 | 选择"企业"时显示"社会信用代码"字段 |
| **值联动** | 字段 A 的值变化后自动填充字段 B | 选择"部门"后自动填充"部门负责人" |
| **禁用联动** | 字段 A 的值决定字段 B 是否可编辑 | 勾选"同上"后禁用地址输入 |

#### 显示/隐藏动效

- 使用 CSS `transition` 实现高度过渡，时长 `$duration-normal`（250ms），缓动 `$ease-out`
- 不使用 `v-if`（会导致 DOM 销毁重建闪烁），使用包裹容器控制高度

```vue
<template>
  <view
    class="form-field-animated"
    :style="{ maxHeight: showCreditCode ? '120rpx' : '0' }"
  >
    <wd-cell title="社会信用代码" required>
      <wd-input v-model="form.creditCode" placeholder="请输入" />
    </wd-cell>
  </view>
</template>

<style lang="scss">
.form-field-animated {
  overflow: hidden;
  transition: max-height $duration-normal $ease-out;
}
</style>
```

#### 值联动反馈

- 选择 A 后自动填充 B 时，B 字段短暂高亮：`$color-primary-light` 背景，`$duration-slow`（400ms）后恢复
- 高亮仅闪烁 1 次，不循环

#### 禁用联动样式

- 被联动禁用的字段：背景 `$color-bg-page`，文字色 `$color-text-placeholder`
- 禁用态的 `wd-input` 设置 `disabled` 属性，`wd-picker` 设置 `disabled` 属性

#### 联动强制约束

1. **联动字段的校验跟随显示状态**：隐藏的字段不参与提交校验，显示后恢复校验
2. **联动触发时机**：值变化后立即触发，不等 blur
3. **多级联动**：A → B → C 的链式联动，中间字段变化时清空下游字段的值

### 5.9 长表单分组与分步

#### 分组规范

| 字段数量 | 处理方式 |
|---------|---------|
| ≤ 6 | 可不分组，或按业务语义分 1-2 组 |
| 7 - 12 | 必须分组，每组 3-5 个字段，使用 `PageSection` 包裹 |
| 13 - 20 | 必须分组 + 非首要分组默认折叠 |
| > 20 | 使用分步表单 |

#### 分组折叠

- 使用 `wd-collapse` + `wd-collapse-item` 实现
- 首要分组（第一组）默认展开，其余默认折叠
- 折叠标题右侧显示该组已填/总数（如 "2/4"），使用 `$font-size-sm` + `$color-text-secondary`
- 含校验错误的分组自动展开，标题右侧显示错误数量角标（`$color-danger`）

#### 分步表单

当字段数 > 20 或业务流程天然分阶段时使用：

```
┌─────────────────────────────────────────┐
│  ① 基本信息 ─── ② 详细信息 ─── ③ 确认  │  ← wd-steps，最多 4 步
├─────────────────────────────────────────┤
│                                         │
│  当前步骤的表单字段                       │
│                                         │
├─────────────────────────────────────────┤
│       [ 上一步 ]         [ 下一步 ]      │  ← 底部固定操作栏
└─────────────────────────────────────────┘
```

- 顶部步骤条：使用 `wd-steps`，最多 4 步，当前步高亮 `$color-primary`
- 每步独立校验：当前步校验通过才能进入下一步，未通过时 toast 提示"请完善当前步骤信息"
- 底部操作栏：第一步隐藏"上一步"，最后一步"下一步"变为"提交"
- 步骤间切换使用左右滑动过渡，时长 `$duration-normal`

#### 进度提示

- 长表单（字段数 > 10）顶部显示进度文案："已填 X/Y 项"
- 使用 `$font-size-sm` + `$color-text-secondary`，右对齐于页面标题下方

### 5.10 上传字段规范

#### 基础配置

| 维度 | 规范 |
|------|------|
| **组件** | 统一使用 `wd-upload` |
| **布局** | 上传区域独占一行，左对齐，与 `PageSection` 等宽 |
| **默认最大数量** | 图片：9 张；文件：5 个（可按业务调整） |
| **超出处理** | 达到最大数量后隐藏添加按钮 |
| **预览** | 点击已上传图片调用 `uni.previewImage` 全屏预览 |

#### 限制提示

- 在上传区域下方显示格式和大小限制文案
- 样式：`$font-size-xs`（10px） + `$color-text-secondary`
- 示例："支持 jpg、png 格式，单张不超过 10MB"、"支持 pdf、doc、xlsx，单个不超过 20MB"

#### 上传状态

| 状态 | 视觉处理 |
|------|---------|
| **待上传** | 虚线边框添加按钮，`$color-border` + `$color-text-placeholder` 图标 |
| **上传中** | 缩略图上覆盖半透明蒙层 + 进度百分比文字（居中，`$font-size-sm`，白色） |
| **上传成功** | 正常显示缩略图，无额外标记（克制原则） |
| **上传失败** | 缩略图覆盖半透明蒙层 + 重试图标（居中），边框变为 `$color-danger` |

#### 必填校验

- 必填的上传字段，提交时检查已上传数量 ≥ 1
- 错误提示："请上传${label}"，显示在上传区域下方，`$color-danger` + `$font-size-sm`

#### 1 个按钮

- **适用场景**：所有操作频率接近、或有危险操作需要"藏"起来

| 条件 | 选择 |
|------|------|
| 操作中有一个明显高频（如"审批"） | 模式 A |
| 操作频率相近 | 模式 B |
| 有危险操作（删除）需要降低误触概率 | 模式 B |
| 列表项本身信息密度高，右侧空间紧张 | 模式 B |

| 维度 | 规范 |

| **按钮间距** | `$spacing-xs`（8px） |

| **危险操作** | 始终使用 `$color-danger`，且需 `wd-message-box` 二次确认 |

#### 格式化规则表

| 数据类型 | 匹配关键词 | 格式化规则 | 输出示例 | 空值处理 |
|----------|-----------|-----------|---------|---------|
| **金额** | `amount/price/cost/money/fee/金额/费用` | `¥` + 千分位 + 两位小数 | `¥ 125,000.00` | `¥ 0.00` |

| **日期** | `date/日期` | `YYYY-MM-DD` | `2024-03-15` | `--` |
| **日期时间** | `datetime/createdAt/updatedAt/时间` | `YYYY-MM-DD HH:mm` | `2024-03-15 10:30` | `--` |
| **相对时间**（列表副信息） | 同日期时间 | 7 天内相对时间，超出绝对日期 | `3小时前` / `2024-03-10` | `--` |
| **手机号** | `phone/mobile/手机/电话` | 中间 4 位脱敏 | `138****8888` | `--` |
| **身份证** | `idCard/身份证/证件号` | 后 8 位脱敏 | `110101**********` | `--` |
| **银行卡** | `bankCard/银行卡/卡号` | 每 4 位空格，前 12 位脱敏 | `**** **** **** 8888` | `--` |
| **百分比** | `percent/rate/ratio/百分比/比例` | 保留 1 位小数 + `%` | `12.5%` | `0%` |
| **数量** | `count/num/qty/数量/件数` | 千分位整数 | `1,200` | `0` |
| **布尔值** | `is*/has*/enable*` | `是` / `否` | `是` | `否` |
| **URL** | `url/link/website/网址/链接` | 列表截断显示域名，详情完整可点击 | `example.com` | `--` |
| **空值** | `null/undefined/''` | 统一显示占位符 | `--` | — |

#### 格式化工具函数签名

```typescript
// utils/formatters.ts — Agent 生成表单/列表/详情代码时自动引入
export const formatAmount = (val: number | null, unit?: string): string
export const formatDate = (val: string | null, pattern?: string): string
export const formatRelativeTime = (val: string | null): string
export const formatPhone = (val: string | null): string
export const formatIdCard = (val: string | null): string
export const formatBankCard = (val: string | null): string
export const formatPercent = (val: number | null, decimal?: number): string
export const formatCount = (val: number | null): string
export const formatEmpty = (val: unknown, fallback?: string): string
```

#### Agent 强制约束

1. **禁止模板内联**：`{{ item.amount.toFixed(2) }}` 禁止，必须调用 `formatAmount(item.amount)`
2. **空值统一兜底**：所有字段值渲染前必须经过 `formatEmpty` 兜底，禁止直接渲染 `null` 或 `undefined`
3. **金额必加单位**：金额字段必须显示 `¥` 前缀，不允许裸数字
4. **列表手机号默认脱敏**：列表场景手机号一律脱敏，详情页按权限决定（参见 §7.4）

### 6.7 列表项数据展示智能推断

> 类比 §5.7 表单字段推断，Agent 根据数据字段名自动推断列表项中每个字段的展示方式，无需用户逐一指定。

#### 推断映射表

| 字段名关键词 | 推断展示方式 | 样式规范 |
|-------------|------------|---------|

| `amount` / `price` / `金额` / `费用` / `cost` | 金额格式化，加粗 | 右侧，`$font-weight-bold` + `$color-text-primary`，`formatAmount()` |
| `date` / `time` / `createdAt` / `updatedAt` / `日期` | 相对时间（副信息）/ 绝对日期（主信息） | `$font-size-sm` + `$color-text-secondary`，`formatRelativeTime()` |

#### 推断规则（Agent 强制遵守）

1. **主标题优先**：含 `name`/`title`/`subject`/`名称`/`标题` 的字段自动作为列表项主标题
2. **状态标签靠左**：推断为 `status` 的字段渲染为 `StatusTag`，固定在列表项底部左侧
3. **金额靠右加粗**：推断为金额的字段在列表项右侧显示，`$font-weight-bold`
4. **副信息用 `·` 分隔**：人名、部门、时间等副信息字段用 `·` 连接成一行
5. **最多 4 个副信息字段**：超出省略，优先保留人名和时间
6. **格式化必须调用**：所有推断出的数据类型必须使用 §6.6 对应的格式化函数

1. **金额字段必须加粗放大**：推断为金额的字段，value 必须使用 `$font-weight-bold` + `$font-size-md`
2. **可交互字段必须有视觉提示**：phone/email/url 等可点击字段必须使用 `$color-primary` + 对应图标
3. **敏感字段默认脱敏**：idCard/bankCard/phone 等字段默认脱敏展示，不依赖后端处理
4. **编号字段提供复制**：推断为编号/单号的字段，value 右侧显示复制图标
5. **格式化必须调用**：所有推断出的数据类型必须使用 §6.6 对应的格式化函数

### 7.5 详情页头部规范

> 详情页头部是用户进入页面后的第一视觉焦点，必须在最小空间内传达最关键的信息。

#### 三层信息层级

```
┌─────────────────────────────────────────┐
│ 采购合同-2024001              [已生效] ✓│  ← 第一层：标题 + 状态
│ 合同金额  ¥ 125,000.00                  │  ← 第二层：关键指标
│ 张三 · 采购部 · 2024-03-15 创建         │  ← 第三层：元信息
└─────────────────────────────────────────┘
```

| 层级 | 内容 | 样式 |
|------|------|------|
| **第一层：标题 + 状态** | 业务对象名称 + StatusTag | `$font-size-lg`（18px）+ `$font-weight-bold`，StatusTag 紧跟标题右侧 |
| **第二层：关键指标** | 最多 1-2 个关键数值（金额、数量等） | `$font-size-md`（16px）+ `$font-weight-bold`，金额使用 `$color-primary` |
| **第三层：元信息** | 创建人、部门、时间等 | `$font-size-sm`（12px）+ `$color-text-secondary`，用 `·` 分隔 |

#### 头部容器样式

- 使用 `PageCard` 包裹，内距 `$spacing-base`
- 可选品牌色浅底背景：`$color-primary-light`（仅在需要视觉强调时使用）
- 三层之间间距：第一层与第二层 `$spacing-xs`（8px），第二层与第三层 `$spacing-xxs`（4px）

#### Agent 选择规则

| 条件 | 头部结构 |
|------|---------|
| 有金额字段 | 三层完整：标题 + 金额 + 元信息 |
| 无金额但有关键数值（数量、进度等） | 三层完整：标题 + 数值 + 元信息 |
| 纯信息详情（无数值） | 两层：标题 + 元信息，省略第二层 |
| 有多个关键指标 | 第二层最多 2 个指标并排，用 `$spacing-lg` 间隔 |

### 7.6 附件展示规范

> 详情页中文件和图片的展示规范。使用 `PageSection` 包裹，标题含数量。

#### 图片附件

```
┌─────────────────────────────────────────┐
│ 图片（3）                                │  ← PageSection 标题
│ ┌─────────┐ ┌─────────┐ ┌─────────┐    │
│ │         │ │         │ │         │    │  ← 每行 3 张，wd-img

│ │         │ │         │ │         │    │
│ └─────────┘ └─────────┘ └─────────┘    │
└─────────────────────────────────────────┘
```

| 维度 | 规范 |
|------|------|
| **组件** | `wd-img`，`mode="aspectFill"` |
| **布局** | 每行 3 张，间距 `$spacing-xs`（8px） |
| **尺寸** | 宽度自适应（`(100% - 2 * 8px) / 3`），高度等于宽度（正方形） |
| **圆角** | `$radius-sm`（4px） |
| **预览** | 点击调用 `uni.previewImage`，支持左右滑动浏览 |
| **超出处理** | 图片数量 > 9 时，第 9 张显示 `+N` 蒙层（半透明黑底 + 白色文字） |

#### 文件附件

```
┌─────────────────────────────────────────┐
│ 文件（2）                                │  ← PageSection 标题
│ ┌─────────────────────────────────────┐ │
│ │ 📄  合同扫描件.pdf          2.3 MB  │ │  ← 文件卡片
│ └─────────────────────────────────────┘ │
│ ┌─────────────────────────────────────┐ │

│ └─────────────────────────────────────┘ │
└─────────────────────────────────────────┘
```

| 维度 | 规范 |
|------|------|
| **卡片高度** | `88rpx` |
| **卡片样式** | 圆角 `$radius-sm`，背景 `$color-bg-page`，内距 `$spacing-sm` |
| **图标** | 左侧文件类型图标，`32rpx × 32rpx` |
| **文件名** | `$font-size-base` + `$color-text-primary`，单行截断 |
| **文件大小** | 右侧，`$font-size-sm` + `$color-text-secondary` |
| **点击操作** | `uni.downloadFile` + `uni.openDocument` 预览/下载 |

#### 文件类型图标映射

| 文件类型 | 图标颜色 | 匹配扩展名 |
|---------|---------|-----------|
| PDF | `#E5252A`（红色） | `.pdf` |
| Word | `#2B5797`（蓝色） | `.doc` / `.docx` |
| Excel | `#207245`（绿色） | `.xls` / `.xlsx` |
| PPT | `#D04423`（橙红） | `.ppt` / `.pptx` |
| 图片 | `$color-primary` | `.jpg` / `.png` / `.gif` |
| 其他 | `$color-text-secondary`（灰色） | 其余所有类型 |

#### 混合附件

- 图片和文件分开展示，图片在上，文件在下
- 分别使用 `PageSection`，标题 "图片（X）" 和 "文件（X）"

### 7.7 关联数据规范

> 详情页中嵌套的关联列表（如合同下的付款记录、审批关联文件、操作日志等）。

#### 结构示意

```
┌─────────────────────────────────────────┐
│ 付款记录（5）                            │  ← PageSection 标题含数量
│ ┌─────────────────────────────────────┐ │
│ │ 第一期付款        ¥ 50,000.00       │ │  ← 简洁型列表项
│ │─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─│ │  ← hairline 分割线
│ │ 第二期付款        ¥ 30,000.00       │ │
│ │─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─│ │

│ └─────────────────────────────────────┘ │
│              查看全部 5 条 >             │  ← 超出 3 条时显示

```

#### 规范明细

| 维度 | 规范 |
|------|------|
| **容器** | `PageSection` 包裹，标题含数量（如 "付款记录（5）"） |
| **列表项布局** | 使用简洁型变体（§6.8），`wd-cell` 实现 |
| **分割线** | 关联列表项间使用 `hairline` 分割线（区别于主列表的间距分隔） |
| **最大展示数** | 默认最多展示 3 条 |
| **查看全部** | 超出 3 条时底部显示 "查看全部 X 条 >" 链接，`$color-primary` + `$font-size-sm`，点击跳转独立列表页 |

| **关联项点击** | 点击跳转关联详情页（如点击付款记录跳转付款详情） |

#### Agent 强制约束

1. **标题必须含数量**：关联数据的 `PageSection` 标题必须包含总数量
2. **最多 3 条**：默认只展示前 3 条，不允许在详情页内展示完整长列表
3. **查看全部必须跳转**：超出 3 条时必须提供 "查看全部" 入口，跳转独立列表页
4. **数据格式化**：关联列表项中的字段同样遵循 §6.6 格式化规则

### 8.2 提示与确认

| 场景 | 组件 | 说明 |
|------|------|------|
| 操作成功 | `wd-toast` type=success | "提交成功"、"保存成功" |
| 操作失败 | `wd-toast` type=error | "提交失败，请重试" |
| 警告提示 | `wd-toast` type=warning | "网络异常" |
| 危险确认 | `wd-message-box` | 删除、驳回等不可逆操作必须二次确认 |
| 信息确认 | `wd-message-box` | "确定提交审批？" |

#### 反馈文案规范

| 文案类型 | 格式 | 正例 | 反例 |
|---------|------|------|------|
| **成功** | `{动词}成功` | "提交成功"、"删除成功"、"保存成功" | "操作成功"、"成功" |
| **失败** | `{动词}失败，{原因/建议}` | "提交失败，请检查网络后重试" | "操作失败"、"失败" |
| **确认** | `确定{动词}{对象}？` | "确定删除该合同？"、"确定驳回此审批？" | "确定操作？"、"确定？" |
| **网络错误** | 统一文案 | "网络异常，请检查网络后重试" | "系统错误"、"未知错误" |
| **服务端错误** | 优先后端 message，兜底通用文案 | "服务暂时不可用，请稍后重试" | "500 Internal Server Error" |

#### Toast 时长规范

| 类型 | 时长 | 说明 |
|------|------|------|
| `success` | `1500ms` | 成功反馈，快速消失不阻断操作 |
| `error` | `2000ms` | 错误需要用户注意和阅读 |
| `warning` | `2000ms` | 警告需要用户注意 |

#### 禁止使用的文案（Agent 强制遵守）

| 禁止文案 | 应替换为 |
|---------|---------|
| "操作成功" | 具体化："提交成功"、"保存成功"、"删除成功" |
| "操作失败" | 说明原因："提交失败，请重试" |
| "系统错误" | "服务暂时不可用，请稍后重试" |
| "未知错误" | "操作异常，请稍后重试" |
| "确定？" | 包含对象："确定删除该合同？" |

### 8.5 弹窗与浮层

> 统一弹窗、抽屉、动作面板等浮层类组件的使用规则。确认弹窗参见 §8.2，动效参见 §8.4。

#### 组件选择决策表

| 场景 | 推荐组件 | position | 说明 |

#### Agent 选择规则

| 条件 | 选择 |

| 侧边滑出的筛选/设置面板 | `wd-popup` position="right" |
| 列表项的更多操作（≤5 项） | `wd-popover` |
| 堆叠弹窗（弹窗上再弹弹窗） | 禁止。重新设计交互流程，避免超过 1 层弹窗 |

### 8.6 图片处理

> 规范图片组件的加载态、错误态、懒加载策略和格式降级。

#### 场景与组件映射

| 场景 | 组件 | 关键属性 | 说明 |
|------|------|----------|------|
| 头像 | `wd-img` | `round`, `width`, `height` | 圆形裁切 |
| 列表缩略图 | `wd-img` | `mode="aspectFill"`, `radius` | 固定宽高比 |
| 详情大图 | `wd-img` | `enable-preview` | 支持点击预览 |
| 轮播图 | `wd-swiper` | — | 首页 Banner 等场景 |
| 上传预览 | `wd-upload` | — | 带删除操作，参见 §5.4 |

#### 加载态与错误态

| 维度 | 规范 |
|------|------|
| **加载中** | 使用 `wd-img` 的 loading slot，展示与图片同尺寸的骨架色块（`$color-bg-page`） |
| **加载失败** | 使用 `wd-img` 的 error slot，展示灰色占位图标 + "加载失败" 文案 |
| **占位背景** | `$color-bg-page`（#F5F6FA） |
| **圆角** | 跟随容器圆角 Token（头像用 `50%`，卡片用 `$radius-md`） |
| **过渡** | 图片加载完成后 fade-in，`$duration-fast`（150ms） |

#### 懒加载策略

| 条件 | 方案 |
|------|------|
| 列表中 ≤10 张图片 | 直接加载，无需懒加载 |
| 列表中 >10 张图片 | 使用 `uni.createIntersectionObserver` 实现视口懒加载 |
| **触发阈值** | 图片距视口底部 `100px` 时开始加载 |
| **首屏图片** | 首屏可见图片不做懒加载，直接加载 |

#### WebP 降级

- CDN 图片优先请求 WebP 格式（如 `?x-oss-process=image/format,webp`）
- 首次通过 `uni.getSystemInfoSync()` 检测 WebP 支持，结果缓存至应用级变量
- 不支持 WebP 时回退原格式（PNG/JPEG）
- 空状态占位图使用本地 PNG，不走 CDN 格式转换

### 8.7 长列表性能

> 不同数据量级下的列表渲染策略。列表结构参见 §6.1，加载更多参见 §6.5。

#### 数据量决策表

| 数据量 | 推荐方案 | 实现方式 | 说明 |
|--------|----------|----------|------|
| ≤50 条 | 直接渲染 | `wd-loadmore` 无限滚动 | 参见 §6.5 |
| 50-200 条 | 分页加载 | 每页 20 条 + `wd-loadmore` | 滚动到底自动加载下一页 |
| 200-1000 条 | 虚拟滚动 | App 端 `<recycle-list>`，H5/小程序自定义虚拟列表 | 仅渲染可视区 + 缓冲区 |
| >1000 条 | 服务端分页 | 强制每页 ≤20 条，提供搜索/筛选缩小范围 | 禁止一次性拉取全量 |

#### 无限滚动 vs 分页选择

| 条件 | 选择 |
|------|------|
| 浏览型场景（动态流、消息列表） | 无限滚动 |
| 需要跳转到特定位置 | 分页 |
| 数据有明确总量且用户关心总数 | 分页（展示"第 X/Y 页"） |
| H5 需 SEO | 分页（URL 携带页码） |

#### 虚拟滚动规范

| 维度 | 规范 |
|------|------|
| **列表项高度** | 优先固定高度；动态高度时使用预估值 + 滚动后修正 |
| **缓冲区** | 可视区上下各 5 条 |
| **占位** | 未渲染区域使用等高占位元素，保持滚动条准确 |
| **scroll-to-index** | 虚拟列表需支持跳转到指定索引 |

#### 性能预算

| 指标 | 目标值 |
|------|--------|
| 首屏渲染条目 | ≤20 条 |
| 可视区 DOM 节点 | ≤30 个列表项 |
| 滚动帧率 | 60fps（单帧 ≤16ms） |

### 8.10 异常状态（离线 / 弱网 / 错误页面）

> 统一全页级异常的检测、展示和恢复策略。空状态占位图参见 `empty-state-assets.md`，Toast 反馈参见 §8.2。

#### 异常类型决策矩阵

| 异常类型 | 检测方式 | 展示方案 | 用户恢复动作 |
|----------|----------|----------|-------------|
| 设备离线 | `uni.getNetworkType()` 返回 `none` | 全页 `wd-status-tip` image="network" | "检查网络设置" 按钮 |
| 弱网超时 | 请求超时 >5s | `wd-toast` type=warning "网络较慢" | 保留已有缓存数据，后台重试 |
| API 400-499 | HTTP 状态码 | `wd-toast` type=error + 具体文案 | 修正输入后重新提交 |
| 资源不存在（404） | HTTP 404 | 全页 `wd-status-tip` image="content" | "返回上一页" 按钮 |
| 服务端错误（5xx） | HTTP 500-599 | 全页 `wd-status-tip` image="network"，使用 `server-busy.png` | "稍后重试" 按钮 |
| 登录过期（401） | HTTP 401 | 跳转登录页 | 重新登录后自动返回原页面 |
| 小程序版本更新 | `uni.getUpdateManager()` | `wd-message-box` 提示重启 | 确认后重启应用 |

#### 全页错误布局

```
┌─────────────────────────────────────────┐
│ ←  页面标题                              │
├─────────────────────────────────────────┤
│                                         │
│              ┌─────────┐                │
│              │  占位图  │                │  ← wd-status-tip / EmptyState
│              └─────────┘                │
│           页面出了点问题                  │  ← $font-size-md + $color-text-primary
│        请稍后再试或检查网络               │  ← $font-size-sm + $color-text-secondary
│                                         │
│            [  重新加载  ]                │  ← $color-primary 主按钮
│                                         │
└─────────────────────────────────────────┘
```

#### 重试策略

| 维度 | 规范 |
|------|------|
| **自动重试** | 最多 1 次，静默执行（用户无感知） |
| **手动重试** | 全页错误提供"重新加载"按钮，列表加载失败提供"点击重试" |
| **重试间隔** | 指数退避：1s → 3s → 9s |
| **缓存优先** | 有缓存数据时优先展示缓存 + 顶部 `wd-notify` 提示"离线模式 - 显示缓存数据" |

#### 强制约束

1. **禁止白屏**：任何异常都不允许展示空白页面，必须有占位图 + 文案 + 恢复操作
2. **禁止暴露技术细节**：用户看到的是"页面出了点问题"，而非"500 Internal Server Error"
3. **必须提供出口**：每个错误页面至少提供一个操作按钮（重试/返回/回首页）

### 8.11 权限与角色差异化

> 不同用户角色看到的页面差异处理规范。复杂度信号 `role_sensitive` 参见 `command-contract.yaml`。

#### 差异化策略表

| 差异类型 | 实现方式 | 适用场景 |
|----------|----------|----------|
| **模块隐藏** | `v-if` 条件渲染 | 整个 `PageSection` 对某角色不可见 |
| **按钮隐藏** | `v-if` 条件渲染 | 管理员可见"删除"，普通用户不可见 |
| **按钮禁用** | `disabled` + 提示说明 | 功能存在但当前角色无权限，需让用户知道 |
| **字段脱敏** | 显示 `***` + 可选"查看"按钮 | 手机号、身份证号等敏感字段 |
| **导航过滤** | 按角色过滤菜单项 | Tabbar、侧边栏按角色显隐 |

#### 双角色对比示意

```
┌─── 普通员工视角 ───┐  ┌──── 管理员视角 ────┐
│ 合同详情            │  │ 合同详情            │
│ 名称  XXX合同       │  │ 名称  XXX合同       │
│ 金额  ¥125,000     │  │ 金额  ¥125,000     │
│                     │  │ 成本  ¥80,000  ← 额外 │
│ ┌─────────────────┐ │  │ ┌─────────────────┐ │
│ │    [  查 看  ]   │ │  │ │[删除] [编辑] [查看]│ │
│ └─────────────────┘ │  │ └─────────────────┘ │
└─────────────────────┘  └─────────────────────┘
```

#### 实现规范

| 维度 | 规范 |
|------|------|
| **权限数据来源** | 登录接口返回权限列表，存入 Pinia/Vuex 全局状态 |
| **权限判断** | 封装 `hasPermission(code)` 工具函数，模板中使用 `v-if="hasPermission('xxx')"` |
| **权限加载中** | 使用 `wd-skeleton` 骨架屏（参见 §8.3），禁止闪烁（先显示再隐藏） |
| **无权限页面** | 禁止展示空白页。使用 `wd-status-tip` + "暂无权限" 文案 + "联系管理员"引导 |

#### Agent 选择规则

| 条件 | 选择 |
|------|------|
| 不同角色看到的布局完全不同 | `v-if` 条件渲染，按角色渲染不同模板 |
| 同一布局，部分元素显隐 | `v-if` 隐藏无权限元素 |
| 用户应知道功能存在但无权使用 | `disabled` 禁用态 + 解释文案 |
| 敏感数据字段 | 默认脱敏显示，授权角色可点击"查看原文" |
| CSS `display:none` 隐藏权限内容 | 禁止。必须用 `v-if` 避免 DOM 泄漏敏感信息 |

### 8.12 页面导航与转场

> 统一列表、详情、表单三种页面类型之间的导航方式、数据传递和返回刷新策略。

#### 导航路径

| 路径 | 导航方式 | 参数传递 | 说明 |
|------|---------|---------|------|
| 列表 → 详情 | `uni.navigateTo` | query 传 `id` | 详情页自行请求数据，不传完整对象 |
| 列表 → 新建表单 | `uni.navigateTo` | query 传 `mode=create` | 表单页空态 |
| 详情 → 编辑表单 | `uni.navigateTo` | query 传 `id` + `mode=edit` | 表单页请求数据回填 |
| 表单提交成功 → 返回 | `uni.navigateBack` | 通过事件通知 | 上一页刷新数据 |

#### 参数传递规范

```typescript
// 列表 → 详情：query 传 id
uni.navigateTo({ url: `/pages/contract/detail?id=${item.id}` })

// 详情 → 编辑：query 传 id + mode
uni.navigateTo({ url: `/pages/contract/form?id=${id}&mode=edit` })

// 表单提交成功 → 通知列表刷新
uni.$emit('list-refresh', { source: 'contract-form' })
uni.navigateBack()
```

#### 返回刷新策略

| 场景 | 策略 | 实现方式 |
|------|------|---------|
| **表单提交后列表刷新** | 事件通知 | 表单页 `uni.$emit('list-refresh')`，列表页 `onShow` 中 `uni.$on` 监听并刷新 |
| **详情页操作后列表刷新** | 同上 | 详情页操作（审批/删除）后 `uni.$emit('list-refresh')` |
| **列表页返回保持状态** | `onShow` 恢复 | 列表页将滚动位置和筛选状态存入组件 data，`onShow` 时恢复 |
| **筛选状态保持** | 组件 data 持久化 | 筛选条件存在 `ref` 中，页面未销毁时自动保持 |

#### 事件通信规范

```typescript
// 列表页 — 监听刷新事件
onShow(() => {
  // 页面重新显示时检查是否需要刷新
})

onLoad(() => {
  uni.$on('list-refresh', () => {
    loadList(true) // 重置并重新加载
  })
})

onUnload(() => {
  uni.$off('list-refresh') // 页面销毁时移除监听
})
```

#### Agent 强制约束

1. **禁止 query 传大对象**：只传 `id`，详情/表单页自行请求完整数据
2. **必须清理事件监听**：`onLoad` 中 `uni.$on`，`onUnload` 中 `uni.$off`，防止内存泄漏
3. **表单提交后必须通知刷新**：不能只 `navigateBack`，必须同时 `uni.$emit` 通知上一页
4. **编辑模式必须标识**：表单页通过 `mode` 参数区分新建和编辑，编辑模式需请求数据回填

### 8.13 全局错误处理

> 与 §8.10 异常状态互补：§8.10 管离线/弱网/错误页面等系统级异常，§8.13 管业务操作级错误。

#### 业务操作错误处理表

| 场景 | 处理方式 | 用户输入 | 按钮状态 |
|------|---------|---------|---------|
| **表单提交失败** | `wd-toast` type=error，显示具体错误 | 保留不清空 | 恢复可点击 |
| **表单提交超时** | `wd-toast` "网络超时，请重试" | 保留不清空 | 恢复可点击 |
| **列表首次加载失败** | `EmptyState` type=load-failed + "重试"按钮 | — | 点击重试 |
| **列表加载更多失败** | `wd-loadmore` state=error | — | 点击重试加载下一页 |
| **详情页 404** | `EmptyState`，文案"内容不存在或已删除" + "返回"按钮 | — | 点击返回上一页 |
| **详情页权限不足** | `EmptyState`，文案"暂无权限查看" + "返回"按钮 | — | 点击返回上一页 |
| **删除操作失败** | `wd-toast` "删除失败，请重试" | — | 不改变列表状态 |
| **审批操作失败** | `wd-toast` 显示具体错误 | — | 恢复可点击 |

#### 错误处理代码模板

```typescript
// 表单提交 — 保留输入，恢复按钮
const submitting = ref(false)

async function handleSubmit() {
  submitting.value = true
  try {
    await api.submit(form.value)
    uni.showToast({ title: '提交成功', icon: 'success', duration: 1500 })
    uni.$emit('list-refresh')
    uni.navigateBack()
  } catch (err) {
    // 不清空 form.value，保留用户输入
    uni.showToast({
      title: err.message || '提交失败，请重试',
      icon: 'error',
      duration: 2000
    })
  } finally {
    submitting.value = false // 恢复按钮
  }
}
```

```typescript
// 列表加载 — 区分首次和加载更多
const list = ref([])
const loading = ref(true)
const loadError = ref(false)
const loadMoreState = ref<'loading' | 'finished' | 'error'>('loading')

async function loadList(reset = false) {
  if (reset) { list.value = []; loadError.value = false }
  loading.value = list.value.length === 0
  try {
    const res = await api.getList({ page, size: 20 })
    list.value = reset ? res.data : [...list.value, ...res.data]
    loadMoreState.value = res.data.length < 20 ? 'finished' : 'loading'
  } catch (err) {
    if (list.value.length === 0) {
      loadError.value = true // 首次加载失败 → EmptyState
    } else {
      loadMoreState.value = 'error' // 加载更多失败 → loadmore error
    }
  } finally {
    loading.value = false
  }
}
```

#### 错误文案映射

| 错误类型 | 用户友好文案 |
|---------|------------|
| 网络超时 (`timeout`) | "网络超时，请重试" |
| 网络断开 (`network error`) | "网络异常，请检查网络后重试" |
| 401 未授权 | "登录已过期，请重新登录"（跳转登录页） |
| 403 无权限 | "暂无权限执行此操作" |
| 404 不存在 | "内容不存在或已删除" |
| 500 服务端错误 | 优先显示后端 `message`，兜底 "服务暂时不可用，请稍后重试" |
| 其他未知错误 | "操作异常，请稍后重试" |

#### Agent 强制约束

1. **表单提交必须 try/catch**：失败时不清空用户输入，按钮 `finally` 中恢复
2. **列表请求必须区分首次和加载更多**：首次失败用 EmptyState，加载更多失败用 loadmore error
3. **按钮操作前必须设 loading**：防止重复点击，`finally` 中恢复
4. **禁止吞掉错误**：catch 中必须给用户反馈（toast 或 EmptyState），不能空 catch
5. **401 统一跳登录**：检测到 401 时跳转登录页，不在业务页面处理
