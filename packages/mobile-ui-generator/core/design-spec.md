# 移动端 UI 设计规范 v5.0

> 适用范围：微信小程序 + H5 | 技术栈：uni-app + Vue 3 + UnoCSS + SCSS
> 组件库：wot-design-uni (小程序+H5, 前缀 `wd-`) / Vant 4 (仅H5, 前缀 `van-`)
> 主题配置：`theme.json`（单源真理） | Token 文件：`core/tokens/`

---

## 0. 设计哲学

> **核心理念：克制而不传统，新颖而不花里胡哨。**

| 原则 | 说明 | 正例 | 反例 |
|------|------|------|------|
| **信息克制** | 每个区块只传达一个核心信息，拒绝堆砌 | 统计卡片只展示数字 + 趋势 | 卡片内同时放数字、图表、说明文字、操作按钮 |
| **留白即设计** | 用间距而非装饰表达层次 | 模块间 `$spacing-lg`（24px） | 模块间用分割线 + 色块 + 图标分隔 |
| **功能驱动装饰** | 装饰元素服务于信息架构，仅一级页面头部允许品牌化元素 | 头部渐变 + 低透明度几何装饰引导视觉焦点 | 二级页面也加渐变头部和插图 |
| **微交互代替大动效** | `$duration-fast`（150ms）的按钮反馈 > 花哨的 Lottie | 按钮按压态 0.95 缩放 + 颜色变深 | 页面加载播放 2 秒品牌动画 |
| **一致性 > 个性化** | 所有页面共享同一套 Token，不允许页面级特殊处理 | 所有卡片统一 `$radius-md` | "这个页面特殊一点，用 12px 圆角" |
| **组件优先** | 优先使用 `wot-design-uni` 标准组件，仅在组件无法满足时自定义 | `wd-cell-group` 实现表单 | 手写 div 模拟表单布局 |
| **真实感优先** | 设计应像有经验的人类设计师手工打磨，拒绝 AI 批量生成的模板感 | 首屏有明确视觉重心，区块有大小主次之分，间距有节奏变化 | 所有卡片等大等高、2×2 网格完美对称、渐变头部 + 圆形图标金刚区的万能模板 |

## 1. Design Tokens

Token 体系分三层：基础层 → 语义层 → 主题层。

- **基础层**：`core/tokens/base-tokens.scss` — 原始色值、字号、间距等
- **语义层**：`core/tokens/semantic-tokens.scss` — CSS Custom Properties 映射
- **主题层**：`core/tokens/theme-presets/*.json` — 可切换的主题配置
- **类型契约**：`core/tokens/theme-contract.ts` — TypeScript 类型定义

## 2. 布局规范

### 2.1 页面容器
- 页面底色固定为 `$color-bg-page` (`#F5F6FA`)。
- 内容区域使用 `safe-area` (`px-16px`)。

### 2.2 卡片布局
- 背景：`#FFFFFF`。
- 圆角：标准页 `8px`，"我的"页 `18px`。
- 边框：**默认无边框**。
- 阴影：`0 2px 8px rgba(0,0,0,0.06)`（仅用于 `card-shadow`）。

### 2.3 筛选器 (Filter Pill)
- 激活态：`bg-#2563EB text-white`。
- 非激活态：`bg-#F3F4F6 text-#475569`。

### 2.4 微信小程序平台约束

微信小程序的胶囊按钮（右上角）会占用导航栏区域，一级页面使用自定义导航栏时**必须**适配。

#### 胶囊区布局示意

```
┌─────────────────────────────────────────┐
│ 状态栏 (statusBarHeight)                 │
├──────────────────────────────┬──────────┤
│                              │ ┌──────┐ │
│   自定义导航内容              │ │ 胶囊  │ │
│   (标题/搜索/问候语)          │ │ 按钮  │ │
│                              │ └──────┘ │
├──────────────────────────────┴──────────┤
│ 页面内容区域                              │
└─────────────────────────────────────────┘
```

#### 安全高度计算

```typescript
// 获取胶囊按钮位置
const menuButton = wx.getMenuButtonBoundingClientRect()
// 状态栏高度
const { statusBarHeight } = wx.getSystemInfoSync()

// 自定义导航栏高度（推荐固定计算方式）
const navBarHeight = (menuButton.top - statusBarHeight) * 2 + menuButton.height
// 导航栏总高度 = 状态栏 + 导航栏
const totalNavHeight = statusBarHeight + navBarHeight
```

#### 适配规则

| 规则 | 说明 |
|------|------|
| **右侧安全区** | 导航栏内容右边距 ≥ 胶囊宽度 + `12px` |
| **高度对齐** | 自定义导航栏标题/图标垂直居中于胶囊区域 |
| **贯穿式头部** | 一级页面头部背景从状态栏顶部延伸到内容区，自定义导航栏叠加其上，背景不被截断 |
| **H5 降级** | H5 环境无胶囊，导航栏高度固定 `44px`，顶部取 `var(--status-bar-height, 0px)` |

#### UnoCSS Shortcut 建议

```typescript
// 已有的 safe-area 处理底部
'safe-area': 'px-16px pb-[env(safe-area-inset-bottom)]'
// 建议新增：导航栏安全右间距
'nav-safe-right': 'pr-[calc(var(--mug-capsule-width,87px)+12px)]'
```

## 5. 表单设计规范

### 5.1 表单布局

使用 `wd-cell-group` 作为表单容器，`wd-cell` 作为字段行。

```
┌─────────────────────────────────────────┐
│ 基本信息                          [必填] │  ← PageSection 标题
├─────────────────────────────────────────┤
│ * 合同名称              请输入合同名称   │  ← wd-cell + wd-input
│─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─│  ← hairline 分割线
│ * 合同类型                      请选择 ▸│  ← wd-cell + wd-picker
│─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─│
│   备注说明              请输入备注信息   │  ← 非必填，无星号
├─────────────────────────────────────────┤
│                                         │
│ 签署信息                                 │  ← 第二个 PageSection
├─────────────────────────────────────────┤
│ * 签署日期              请选择日期     ▸│
│─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─│
│ * 签署方                请选择签署方   ▸│
└─────────────────────────────────────────┘
         ...（更多字段）...
┌─────────────────────────────────────────┐
│          [ 保存草稿 ]    [ 提 交 ]       │  ← 底部固定操作栏
└─────────────────────────────────────────┘
```

### 5.2 字段规范

| 维度 | 规范 |
|------|------|
| **字段高度** | 正常模式 `88rpx`，紧凑模式 `72rpx` |
| **标签（label）** | **左对齐**（`text-align: left`），`$font-size-base`，`$color-text-regular`，宽度建议 `180rpx` |
| **输入区** | **右对齐**（`text-align: right`），`$font-size-base`，`$color-text-primary` |
| **占位符** | **右对齐**（与输入区对齐），`$color-text-placeholder` |
| **必填标记** | label 前红色星号 `*`，使用 `$color-danger` |
| **分割线** | 组内使用 `hairline` 分割线（`$color-border-light`），组间使用 `$spacing-lg` 留白 |
| **分组** | 相关字段用 `PageSection` 包裹，组标题用 `$font-size-md` + `$font-weight-medium` |

### 5.3 校验规则

| 维度 | 规范 |
|------|------|
| **触发时机** | 字段失焦（blur）时校验，提交时全量校验 |
| **错误提示** | 字段下方显示，**右对齐**（与输入值对齐），`$color-danger` + `$font-size-sm`（12px） |
| **错误态** | 字段右侧或底部边框变为 `$color-danger` |
| **成功态** | 不做视觉反馈（克制原则，减少干扰） |
| **提交按钮** | 未通过校验时使用 `$color-primary-disabled`，禁止点击 |

### 5.4 表单类型适配

| 字段类型 | 推荐组件 | 说明 |
|----------|----------|------|
| 文本输入 | `wd-input` | 单行文本 |
| 多行文本 | `wd-textarea` | 自动增高，最大 `200rpx` |
| 选择器 | `wd-picker` / `wd-select-picker` | 点击后弹出选择面板 |
| 日期选择 | `wd-datetime-picker` | 年-月-日 / 年-月-日 时:分 |
| 开关 | `wd-switch` | 布尔值切换 |
| 上传 | `wd-upload` | 图片/文件上传，带预览 |
| 评分 | `wd-rate` | 星级评分 |

### 5.5 移动端/小程序表单对齐统一规范

> 适用范围：微信小程序 + H5，所有表单页面强制遵循。

#### 对齐总则

移动端表单采用 **「左 label — 右 value」** 的经典双栏对齐布局，确保视觉一致性和可读性。

```
┌─────────────────────────────────────────┐
│ * 合同名称              采购服务合同-年度│  ← label 左对齐 | value 右对齐
│─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─│
│ * 合同类型                      请选择 ▸│  ← label 左对齐 | placeholder 右对齐
│─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─│
│ * 合同金额                              │
│                        请输入合同金额    │  ← placeholder 右对齐
│                   金额必须大于0          │  ← 校验提示右对齐，与值对齐
└─────────────────────────────────────────┘
```

#### 对齐规则明细

| 元素 | 对齐方式 | 说明 |
|------|----------|------|
| **label（标签）** | `text-align: left` | 始终靠左，包含必填星号 `*` 时星号在 label 最前方 |
| **value（表单值）** | `text-align: right` | 输入值、已选值、显示值统一靠右 |
| **placeholder** | `text-align: right` | 与 value 位置一致，用户输入后无缝替换 |
| **校验错误提示** | `text-align: right` | 显示在字段下方，与 value 区域右对齐 |
| **必填星号** | 跟随 label 左对齐 | 位于 label 文字最前方，使用 `$color-danger` |

#### CSS 实现参考

```scss
// 表单对齐核心样式
.form-field {
  // label 左对齐
  :deep(.wd-cell__title) {
    text-align: left;
  }

  // value、placeholder 右对齐
  :deep(.wd-cell__value),
  :deep(.wd-input__inner),
  :deep(.wd-textarea__inner) {
    text-align: right;
  }

  // placeholder 右对齐（通过 input 的属性传递）
  :deep(.wd-input__inner)::placeholder,
  :deep(.wd-textarea__inner)::placeholder {
    text-align: right;
  }

  // 校验错误提示右对齐
  .form-error-msg {
    text-align: right;
    color: $color-danger;
    font-size: $font-size-sm;
    padding-right: $spacing-base;
    margin-top: $spacing-xxs;
  }
}
```

#### 特殊字段对齐处理

| 字段类型 | 对齐说明 |
|----------|----------|
| **单行文本** (`wd-input`) | value + placeholder 右对齐 |
| **多行文本** (`wd-textarea`) | 独占一行时左对齐（因内容较长），内嵌 cell 时右对齐 |
| **选择器** (`wd-picker`) | 已选值右对齐，placeholder 右对齐，右箭头 `▸` 紧跟其后 |
| **日期选择** (`wd-datetime-picker`) | 同选择器规则 |
| **开关** (`wd-switch`) | 组件本身靠右放置 |
| **上传** (`wd-upload`) | 独占区域，左对齐展示（因需要预览空间） |
| **评分** (`wd-rate`) | 组件靠右放置 |

#### 校验提示对齐示意

```
┌─────────────────────────────────────────┐
│ * 手机号码                  1381234567  │  ← 正常态：value 右对齐
│─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─│
│ * 邮箱地址                  abc@invalid │  ← 错误态：value 右对齐
│                     请输入正确的邮箱格式 │  ← 错误提示：右对齐，与 value 对齐
│─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─│
│ * 合同金额                              │  ← 空值错误态
│                        请输入合同金额    │  ← 错误提示：右对齐
└─────────────────────────────────────────┘
```

#### 强制约束（Agent 必须遵守）

1. **不允许 label 右对齐或居中**：label 始终 `text-align: left`
2. **不允许 value 左对齐**：表单值始终 `text-align: right`（多行文本独占行除外）
3. **placeholder 必须与 value 同侧**：确保用户输入前后视觉位置一致
4. **校验提示必须与 value 对齐**：错误信息出现在用户视线焦点附近，便于定位问题字段
5. **跨端一致**：微信小程序和 H5 端必须表现一致，不允许端差异导致对齐不同

### 6.4 列表操作按钮

这是列表页面中**最常见也最容易出问题的部分**。按钮数量不同，处理方式不同：

#### 1 个按钮

```
│ 合同名称-2024001                        │
│ 张三 · 采购部 · 2024-03-15              │
│ [待审批]                        [审批]  │
```
- 右侧内联文字按钮，使用 `$color-primary`，无底色无边框
- 字号 `$font-size-sm`（12px）

#### 2 个按钮

```
│ 合同名称-2024001                        │
│ 张三 · 采购部 · 2024-03-15              │
│ [待审批]                 [查看] [审批]  │
```
- 右侧并排，主操作（右侧）用 `$color-primary`，次操作（左侧）用 `$color-text-secondary`
- 按钮间距 `$spacing-xs`（8px）

#### 3+ 个按钮 — 模式 A：内联 + 更多菜单

```
│ 合同名称-2024001                        │
│ 张三 · 采购部 · 2024-03-15              │
│ [待审批]                 [审批]  [ ··· ]│

                                    └─────┘
```
- 展示最高频 1 个操作按钮 + `···` 更多菜单（`wd-popover` 或 `wd-action-sheet`）
- **适用场景**：操作频率差异大（1 个高频 + 多个低频）

#### 3+ 个按钮 — 模式 B：左滑操作面板

```
│ 合同名称-2024001            ← 左滑       │
│ 张三 · 采购部                             │

                     └── SwipeCell 展开 ──┘
```
- 使用 `wd-swipe-cell`，左滑展开全部操作按钮
- **适用场景**：所有操作频率接近、或有危险操作需要"藏"起来

#### Agent 选择规则

| 条件 | 选择 |
|------|------|
| 操作中有一个明显高频（如"审批"） | 模式 A |
| 操作频率相近 | 模式 B |
| 有危险操作（删除）需要降低误触概率 | 模式 B |
| 列表项本身信息密度高，右侧空间紧张 | 模式 B |

#### 通用规则

| 维度 | 规范 |
|------|------|
| **按钮尺寸** | mini size，高度 `56rpx`，字号 `$font-size-sm`（12px） |
| **按钮间距** | `$spacing-xs`（8px） |
| **主操作色** | `$color-primary` |
| **次操作色** | `$color-text-secondary`（文字按钮）或 `$color-border`（边框按钮） |
| **危险操作** | 始终使用 `$color-danger`，且需 `wd-message-box` 二次确认 |
| **禁用态** | `$color-text-disabled`，禁止点击 |

### 6.5 列表状态处理

| 状态 | 处理方式 |
|------|----------|
| **首次加载** | 使用 `wd-skeleton` 骨架屏，模拟 3 个列表项 |

| **上拉加载** | `wd-loadmore`，三态：loading / finished / error |

| 场景 | 处理方式 |
|------|----------|
| 页面首次加载 | 骨架屏 `wd-skeleton`，模拟真实布局 |

### 8.6 图片处理

> 规范图片组件的加载态、错误态、懒加载策略和格式降级。

#### 场景与组件映射

| 场景 | 组件 | 关键属性 | 说明 |
|------|------|----------|------|
| 头像 | `wd-img` | `round`, `width`, `height` | 圆形裁切 |
| 列表缩略图 | `wd-img` | `mode="aspectFill"`, `radius` | 固定宽高比 |
| 详情大图 | `wd-img` | `enable-preview` | 支持点击预览 |
| 轮播图 | `wd-swiper` | — | 首页 Banner 等场景 |
| 上传预览 | `wd-upload` | — | 带删除操作，参见 §5.4 |

#### 加载态与错误态

| 维度 | 规范 |
|------|------|
| **加载中** | 使用 `wd-img` 的 loading slot，展示与图片同尺寸的骨架色块（`$color-bg-page`） |
| **加载失败** | 使用 `wd-img` 的 error slot，展示灰色占位图标 + "加载失败" 文案 |
| **占位背景** | `$color-bg-page`（#F5F6FA） |
| **圆角** | 跟随容器圆角 Token（头像用 `50%`，卡片用 `$radius-md`） |
| **过渡** | 图片加载完成后 fade-in，`$duration-fast`（150ms） |

#### 懒加载策略

| 条件 | 方案 |
|------|------|
| 列表中 ≤10 张图片 | 直接加载，无需懒加载 |
| 列表中 >10 张图片 | 使用 `uni.createIntersectionObserver` 实现视口懒加载 |
| **触发阈值** | 图片距视口底部 `100px` 时开始加载 |
| **首屏图片** | 首屏可见图片不做懒加载，直接加载 |

#### WebP 降级

- CDN 图片优先请求 WebP 格式（如 `?x-oss-process=image/format,webp`）
- 首次通过 `uni.getSystemInfoSync()` 检测 WebP 支持，结果缓存至应用级变量
- 不支持 WebP 时回退原格式（PNG/JPEG）
- 空状态占位图使用本地 PNG，不走 CDN 格式转换

### 8.7 长列表性能

> 不同数据量级下的列表渲染策略。列表结构参见 §6.1，加载更多参见 §6.5。

#### 数据量决策表

| 数据量 | 推荐方案 | 实现方式 | 说明 |
|--------|----------|----------|------|
| ≤50 条 | 直接渲染 | `wd-loadmore` 无限滚动 | 参见 §6.5 |
| 50-200 条 | 分页加载 | 每页 20 条 + `wd-loadmore` | 滚动到底自动加载下一页 |
| 200-1000 条 | 虚拟滚动 | App 端 `<recycle-list>`，H5/小程序自定义虚拟列表 | 仅渲染可视区 + 缓冲区 |
| >1000 条 | 服务端分页 | 强制每页 ≤20 条，提供搜索/筛选缩小范围 | 禁止一次性拉取全量 |

#### 无限滚动 vs 分页选择

| 条件 | 选择 |
|------|------|
| 浏览型场景（动态流、消息列表） | 无限滚动 |
| 需要跳转到特定位置 | 分页 |
| 数据有明确总量且用户关心总数 | 分页（展示"第 X/Y 页"） |
| H5 需 SEO | 分页（URL 携带页码） |

#### 虚拟滚动规范

| 维度 | 规范 |
|------|------|
| **列表项高度** | 优先固定高度；动态高度时使用预估值 + 滚动后修正 |
| **缓冲区** | 可视区上下各 5 条 |
| **占位** | 未渲染区域使用等高占位元素，保持滚动条准确 |
| **scroll-to-index** | 虚拟列表需支持跳转到指定索引 |

#### 性能预算

| 指标 | 目标值 |
|------|--------|
| 首屏渲染条目 | ≤20 条 |
| 可视区 DOM 节点 | ≤30 个列表项 |
| 滚动帧率 | 60fps（单帧 ≤16ms） |

### 8.8 数据可视化

> 指标卡、进度组件、表格和图表的移动端规范。复杂度信号 `has_data_visualization` 参见 `command-contract.yaml`。

#### 可视化类型决策表

| 可视化类型 | 推荐方案 | 说明 |
|-----------|----------|------|
| 指标卡（KPI） | 自定义组件 | 数字 + 趋势，无需三方库 |
| 进度条 | `wd-progress` | 线性进度展示 |
| 进度环 | `wd-circle` | 环形进度展示 |
| 简单表格 | `wd-table` | 结构化数据展示 |
| 图表（折线/柱状/饼图） | `ucharts`（小程序+H5） | 跨端优先选择 |
| 迷你趋势线（Sparkline） | 自定义 SVG | 嵌入指标卡内，无需图表库 |

#### 指标卡规范

```
┌──────────────────────┐
│  合同总额          ▸ │  ← $font-size-sm + $color-text-secondary
│  ¥ 1,250,000        │  ← $font-size-hero + $font-weight-heavy
│  ↑ 12.5% 较上月     │  ← 上升 $color-success / 下降 $color-danger
└──────────────────────┘
```

| 维度 | 规范 |
|------|------|
| **标题** | `$font-size-sm`（12px），`$color-text-secondary` |
| **数值** | `$font-size-hero`（26px），`$font-weight-heavy`（700） |
| **趋势** | 上升用 `$color-success`，下降用 `$color-danger`，`$font-size-sm` |
| **卡片** | 背景 `$color-bg-card`，圆角 `$radius-md` |
| **布局** | 单行 1-2 个指标卡，禁止 2×2 等分网格（参见 §8 反 AI 规则） |

#### wd-table 移动端适配

| 维度 | 规范 |
|------|------|
| **可见列数** | 最多 4 列，超出启用横向滚动 |
| **首列固定** | 列数 >4 时首列 `fixed: 'left'` |
| **表头** | 背景 `$color-bg-page`，`$font-size-sm`，`$font-weight-medium` |
| **单元格** | `$font-size-sm`，内距 `$spacing-xs`，最小行高 `72rpx` |
| **斑马纹** | `stripe` 属性启用，颜色 `$color-bg-hover` |
| **边框** | 默认显示横向边框（`$color-border-light`），无纵向边框 |

#### 图表库选择规则

| 条件 | 选择 |
|------|------|
| 目标含小程序 | `ucharts`（原生 Canvas 渲染） |
| 仅 H5 且需复杂交互（缩放、联动） | `echarts` |
| 只需简单趋势线（嵌入卡片） | 自定义 SVG，不引入图表库 |
| 只需进度展示 | `wd-progress` / `wd-circle`，不引入图表库 |

### 8.9 输入法适配

> 键盘弹起时的页面行为规范。表单字段参见 §5.2。

#### 键盘弹起场景处理

| 场景 | 处理方式 | 说明 |
|------|----------|------|
| 普通表单页（可滚动） | 系统自动滚动 | `wd-input` 默认 `adjust-position="true"` |
| 底部固定输入栏（聊天） | 手动跟随键盘 | 监听键盘高度，输入栏 `bottom` 设为键盘高度 |
| 弹窗内表单 | `wd-popup` `lock-scroll="false"` | 确保输入时页面可滚动到焦点位置 |
| 多行文本 | `wd-textarea` `auto-height` | 设置 `max-height` 防止无限增高 |
| 数字输入 | `type="number"` 或 `type="digit"` | 触发数字键盘 |
| 搜索输入 | `confirm-type="search"` | 键盘回车键显示"搜索" |

#### 底部输入栏示意

```
┌─────────────────────────────────────────┐
│  内容区域（向上推移 keyboardHeight）      │
│  ...                                    │
├─────────────────────────────────────────┤
│  [😊]  请输入消息...           [发送]   │  ← 输入栏跟随键盘上移
├─────────────────────────────────────────┤
│  ┌───────────────────────────────────┐  │
│  │           系统键盘区域             │  │
│  └───────────────────────────────────┘  │
└─────────────────────────────────────────┘
```

#### 平台差异

| 平台 | 键盘高度获取方式 | 说明 |
|------|-----------------|------|
| 微信小程序 | `wx.onKeyboardHeightChange` | 实时回调 |
| H5 | `window.visualViewport.resize` 事件 | 通过视口高度变化反推 |
| App | `uni.onKeyboardHeightChange` | 统一 API |

### 8.10 异常状态（离线 / 弱网 / 错误页面）

> 统一全页级异常的检测、展示和恢复策略。空状态占位图参见 `empty-state-assets.md`，Toast 反馈参见 §8.2。

#### 异常类型决策矩阵

| 异常类型 | 检测方式 | 展示方案 | 用户恢复动作 |
|----------|----------|----------|-------------|
| 设备离线 | `uni.getNetworkType()` 返回 `none` | 全页 `wd-status-tip` image="network" | "检查网络设置" 按钮 |
| 弱网超时 | 请求超时 >5s | `wd-toast` type=warning "网络较慢" | 保留已有缓存数据，后台重试 |
| API 400-499 | HTTP 状态码 | `wd-toast` type=error + 具体文案 | 修正输入后重新提交 |
| 资源不存在（404） | HTTP 404 | 全页 `wd-status-tip` image="content" | "返回上一页" 按钮 |
| 服务端错误（5xx） | HTTP 500-599 | 全页 `wd-status-tip` image="network"，使用 `server-busy.png` | "稍后重试" 按钮 |
| 登录过期（401） | HTTP 401 | 跳转登录页 | 重新登录后自动返回原页面 |
| 小程序版本更新 | `uni.getUpdateManager()` | `wd-message-box` 提示重启 | 确认后重启应用 |

#### 全页错误布局

```
┌─────────────────────────────────────────┐
│ ←  页面标题                              │
├─────────────────────────────────────────┤
│                                         │
│              ┌─────────┐                │
│              │  占位图  │                │  ← wd-status-tip / EmptyState
│              └─────────┘                │
│           页面出了点问题                  │  ← $font-size-md + $color-text-primary
│        请稍后再试或检查网络               │  ← $font-size-sm + $color-text-secondary
│                                         │
│            [  重新加载  ]                │  ← $color-primary 主按钮
│                                         │
└─────────────────────────────────────────┘
```

#### 重试策略

| 维度 | 规范 |
|------|------|
| **自动重试** | 最多 1 次，静默执行（用户无感知） |
| **手动重试** | 全页错误提供"重新加载"按钮，列表加载失败提供"点击重试" |
| **重试间隔** | 指数退避：1s → 3s → 9s |
| **缓存优先** | 有缓存数据时优先展示缓存 + 顶部 `wd-notify` 提示"离线模式 - 显示缓存数据" |

#### 强制约束

1. **禁止白屏**：任何异常都不允许展示空白页面，必须有占位图 + 文案 + 恢复操作
2. **禁止暴露技术细节**：用户看到的是"页面出了点问题"，而非"500 Internal Server Error"
3. **必须提供出口**：每个错误页面至少提供一个操作按钮（重试/返回/回首页）

### 8.11 权限与角色差异化

> 不同用户角色看到的页面差异处理规范。复杂度信号 `role_sensitive` 参见 `command-contract.yaml`。

#### 差异化策略表

| 差异类型 | 实现方式 | 适用场景 |
|----------|----------|----------|
| **模块隐藏** | `v-if` 条件渲染 | 整个 `PageSection` 对某角色不可见 |
| **按钮隐藏** | `v-if` 条件渲染 | 管理员可见"删除"，普通用户不可见 |
| **按钮禁用** | `disabled` + 提示说明 | 功能存在但当前角色无权限，需让用户知道 |
| **字段脱敏** | 显示 `***` + 可选"查看"按钮 | 手机号、身份证号等敏感字段 |
| **导航过滤** | 按角色过滤菜单项 | Tabbar、侧边栏按角色显隐 |

#### 双角色对比示意

```
┌─── 普通员工视角 ───┐  ┌──── 管理员视角 ────┐
│ 合同详情            │  │ 合同详情            │
│ 名称  XXX合同       │  │ 名称  XXX合同       │
│ 金额  ¥125,000     │  │ 金额  ¥125,000     │
│                     │  │ 成本  ¥80,000  ← 额外 │
│ ┌─────────────────┐ │  │ ┌─────────────────┐ │
│ │    [  查 看  ]   │ │  │ │[删除] [编辑] [查看]│ │
│ └─────────────────┘ │  │ └─────────────────┘ │
└─────────────────────┘  └─────────────────────┘
```

#### 实现规范

| 维度 | 规范 |
|------|------|
| **权限数据来源** | 登录接口返回权限列表，存入 Pinia/Vuex 全局状态 |
| **权限判断** | 封装 `hasPermission(code)` 工具函数，模板中使用 `v-if="hasPermission('xxx')"` |
| **权限加载中** | 使用 `wd-skeleton` 骨架屏（参见 §8.3），禁止闪烁（先显示再隐藏） |
| **无权限页面** | 禁止展示空白页。使用 `wd-status-tip` + "暂无权限" 文案 + "联系管理员"引导 |

#### Agent 选择规则

| 条件 | 选择 |
|------|------|
| 不同角色看到的布局完全不同 | `v-if` 条件渲染，按角色渲染不同模板 |
| 同一布局，部分元素显隐 | `v-if` 隐藏无权限元素 |
| 用户应知道功能存在但无权使用 | `disabled` 禁用态 + 解释文案 |
| 敏感数据字段 | 默认脱敏显示，授权角色可点击"查看原文" |
| CSS `display:none` 隐藏权限内容 | 禁止。必须用 `v-if` 避免 DOM 泄漏敏感信息 |
