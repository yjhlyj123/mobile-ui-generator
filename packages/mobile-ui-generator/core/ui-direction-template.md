# UI 方案标准化输出模板

> 当进入复杂模式（`/mug:x`）、设计先行（`/mug:d`）或 Tabbar 页面时，
> Agent 必须使用本模板输出至少 3 个差异化 UI 方案。
> 用户未选定前，不得进入结构化 Prompt 或代码实现阶段。

---

## 方案生成原则

在生成方案前，Agent 必须先完成以下思考（不需要输出给用户，但必须体现在方案中）：

1. **用户核心任务**：这个页面的用户打开它，最想完成什么？（如：快速找到待审批项、一眼看到今日数据）
2. **信息优先级**：哪些信息是"必须首屏看到"的，哪些是"需要时再找"的？
3. **操作频率**：哪些操作是高频的（每天多次），哪些是低频的（偶尔用一次）？

三个方案必须在**信息架构**层面有本质差异，而不仅仅是视觉风格不同。具体来说：
- 方案之间的**首屏核心内容**必须不同（如：方案 A 首屏是待办列表，方案 B 首屏是数据概览）
- 方案之间的**用户动线**必须不同（如：方案 A 引导用户先看数据再操作，方案 B 引导用户直接进入高频操作）
- 禁止仅通过换颜色、换圆角、换头部样式来制造"差异化"

## 单方案模板

对每个方案，按以下结构输出：

```md
## 方案 [序号]：[方案名称]

### 一句话定位
用一句话说明：这个方案假设用户打开页面后最想做什么，因此把什么放在最前面。
（示例："假设用户最关心待处理事项，首屏直接展示待办列表，快速进入处理流程。"）

### 用户动线
描述用户从打开页面到完成核心任务的路径：
1. 打开页面 → 首先看到 ___
2. 扫一眼 → 了解到 ___
3. 点击 → 进入 ___

### 布局示意

（ASCII 布局图，展示首屏的核心区块组织。
  要求：标注每个区块承载的具体业务内容，不要只写"模块1"、"区块A"。）

### 首屏信息分配
| 区域 | 承载内容 | 为什么放在这里 |
|------|----------|----------------|
| ___ | ___ | ___ |
| ___ | ___ | ___ |
| ___ | ___ | ___ |

### 主要特征
| 维度 | 处理方式 |
|------|----------|
| 头部 | ___ |
| 导航 | ___ |
| 核心区块 | ___ |
| 色彩策略 | ___ |
| 交互亮点 | ___ |

### 适合这个方案的情况
- 如果你的用户主要是 ___
- 如果最高频的操作是 ___
- 如果最重要的信息是 ___

### 不适合这个方案的情况
- 如果 ___
```

## 方案对比总结模板

在 3 个方案列出后，附加一份对比总结，帮助用户快速做出选择：

```md
## 方案对比总结

| 维度 | 方案 A | 方案 B | 方案 C |
|------|--------|--------|--------|
| 核心假设 | 用户最想 ___ | 用户最想 ___ | 用户最想 ___ |
| 首屏主角 | ___ | ___ | ___ |
| 用户动线 | 先看 → 再做 | 先做 → 再看 | 边看边做 |
| 信息密度 | 高 / 中 / 低 | 高 / 中 / 低 | 高 / 中 / 低 |
| 操作效率 | 适合高频操作 / 适合浏览决策 | ___ | ___ |

### 选择建议

- **选方案 A**：如果 ___（用具体的业务场景描述）
- **选方案 B**：如果 ___
- **选方案 C**：如果 ___

> 请选择一个方案编号（A/B/C），或描述你的偏好方向。
> 也可以混搭："用方案 A 的头部 + 方案 B 的内容区"。
> 选定后我会先用 Pencil 生成视觉设计稿给你预览，你确认设计满意后再进入代码实现。
```

## Tabbar 页面专用扩展

当方案涉及 Tabbar 页面时，在"主要特征"中增加以下行：

```md
| 维度 | 处理方式 |
|------|----------|
| 头部风格 | 沉浸式渐变 / 简洁白底 / 图片背景 |
| 金刚区 | 有 / 无 / 可滚动 |
| 统计卡片 | 有 / 无 |
| 装饰插图 | 有 / 无 |
| Tabbar 样式 | 标准 / 凸起中央按钮 / 带徽标 |
```

## 使用规则

1. **至少 3 个方案**：不得少于 3 个差异化方案
2. **信息架构差异优先**：每个方案的首屏核心内容和用户动线必须不同，视觉差异是附带的
3. **包含布局示意**：每个方案必须有 ASCII 布局图，且图中标注具体业务内容
4. **给出选择建议**：对比总结中必须包含"什么情况选什么"的具体指引
5. **明确等待**：输出方案后必须等待用户选择，禁止自动进入下一步
6. **可迭代**：如果用户不满意，继续提供新方案或微调已有方案
7. **禁止空洞描述**：不得使用"简约大气"、"现代感"、"高端商务"等无信息量的形容词，必须用具体的布局和内容来体现风格
