# Mobile UI Generator Core Pencil Prompt Template

## Standard Template

```md
# 页面信息
- 页面类型：
- 页面层级：一级页面（Tabbar） / 二级页面 / 三级页面
- 业务目标：
- 目标用户：
- 业务场景：

# 主题配置
- 当前主题：（如：商务蓝 / 活力橙 / 清新绿 / 自定义）
- 品牌色：
- 渐变方向：（如：`gradients.headerBg`）

# 核心模块
- 模块 1：
- 模块 2：
- 模块 3：

# 优先级
- 首屏必须表达：
- 次级信息：
- 可省略信息：

# 风格约束
- 遵循 `design-spec.md` 设计规范
- 遵循 `page-hierarchy.md` 页面层级差异
- 颜色通过 CSS Custom Properties 引用
- 信息克制，弱装饰
- 不要营销式头图（二级页面）
- 不要冗余描述文案
- 布局需有节奏变化：大小区块交替，禁止全页等分网格或 2×2 完美对称
- 首屏需有明确视觉重心：一个主角区块 + 辅助区块，不是均匀分布
- 使用真实业务数据作为内容占位（如"¥125,000"、"张三"），不用"标题文本"
- 排版需有明显层次对比：标题与正文字号差 ≥ 4px
- 色彩：大面积留白 + 局部品牌色点睛，不要处处有色彩
- 禁止 AI 套路式布局（2×2 数据网格 + 渐变头部 + 圆形图标金刚区的万能首页模板）

# 技术与落地约束
- uni-app
- Vue 3
- UnoCSS
- wot-design-uni
- CSS Custom Properties (`--mug-*`)

- 参考稿 / `.pen` / 截图：
- 需要保留的元素：
- 需要避免的元素：

# 输出要求
- 给出页面结构
- 给出区块层级
- 给出首屏重点
- 给出自定义导航建议（如需要）
- 给出状态与卡片的视觉方向
```

## Tabbar Page Template (一级页面专用)

Use this template when the page is a primary Tabbar page.

```md
# Tabbar 页面信息
- 页面名称：（如：首页 / 工作台 / 我的）
- 业务目标：
- 目标角色：

# 选定的 Tabbar 风格
- 风格：（沉浸式头部 / 卡片金刚区 / 数据看板式 / 图文并茂式）
- 参考：`tabbar-styles.md` 中的方案描述

# 头部区域
- 背景处理：渐变 + 几何装饰（风格A） / 渐变 + 微弱纹理（风格B）
- 渐变色值：`gradients.headerBg`
- 背景装饰：（描述具体装饰元素，参见 `page-hierarchy.md` § 3.1）
- 头部内容：问候语 / 搜索框 / 通知入口
- 头部高度：约 ___rpx
- 微信胶囊适配：右侧留出胶囊安全距离

# 金刚区（如适用）
- 列数：4 / 5
- 图标风格：渐变圆形底 / 纯色底 / 线性图标
- 入口数量：___

# 统计/概览（如适用）
- 卡片数量：___
- 数据类型：待办 / 金额 / 进度 / 趋势

# 内容分区
- 分区 1：
- 分区 2：
- 分区 3：

# 设计约束
- 使用一级页面规范（大圆角 `$radius-xxl`）
- 头部渐变必须使用主题配置的 `gradients.headerBg`
- 金刚区/统计卡片允许微动效
- 布局需有节奏变化：大小区块交替，禁止全页等分网格或 2×2 完美对称
- 首屏需有明确视觉重心：一个主角区块占据首屏主要面积
- 使用真实业务数据（如"待审批 3 件"、"本月合同金额 ¥52万"），不用"标题文本"
- 禁止 AI 套路：不做"问候语 + 2×2 金刚区 + 列表"万能模板，根据业务场景差异化设计
- 金刚区不是必须的，应根据实际功能入口数量决定是否需要

# 输出要求
- 给出完整的首屏布局层级
- 给出头部与内容的交界处理方式
- 给出 Tabbar 选中态样式建议
```

| 操作效率 | ___ | ___ | ___ |

- **选方案 A**：如果 ___（用具体的业务场景描述）

## Form Page Template (表单页面专用)

Use this template when the page specialty is `form`.

```md
# 表单页面信息
- 页面名称：（如：新建合同 / 提交审批 / 编辑资料）
- 业务目标：
- 目标角色：

# 表单字段清单
| 字段名 | 类型 | 必填 | 组件 | 推断校验类型 | 键盘类型 | 联动说明 |
|--------|------|------|------|-------------|---------|----------|
| ___ | 文本/数字/选择/日期/开关/上传 | 是/否 | wd-input/wd-picker/... | 参照 §5.7 自动推断 | number/digit/idcard/text | ___ |

# 字段分组
- 分组 1 名称：包含字段 ___
- 分组 2 名称：包含字段 ___
- 分组 3 名称：包含字段 ___

# 表单模式
- 字段高度：正常（88rpx） / 紧凑（72rpx）
- 标签宽度：（默认 180rpx）

# 提交操作
- 按钮数量：1（提交） / 2（保存草稿 + 提交）
- 按钮位置：底部固定操作栏
- 提交前校验：全量校验

# 设计约束
- 遵循 `design-spec.md` § 5 表单设计规范
- 使用 `wd-cell-group` + `wd-cell` 布局
- 必填字段标注红色星号
- 校验错误在字段下方显示
- 使用真实业务字段名和占位符（如"请输入合同名称"），不用"请输入内容"
- 字段校验：AI 根据字段名自动推断校验类型和键盘类型，参照 `design-spec.md` §5.7
- 联动动效：字段联动遵循 `design-spec.md` §5.8 联动规范
- 长表单处理：字段分组/折叠/分步遵循 `design-spec.md` §5.9

# 输出要求
- 给出字段分组和排列顺序
- 给出校验交互方式
- 给出提交按钮布局
- 给出联动字段的显示/隐藏逻辑
```

## List Page Template (列表页面专用)

Use this template when the page specialty is `list`.

```md
# 列表页面信息
- 页面名称：（如：合同列表 / 审批记录 / 用章申请）
- 业务目标：
- 目标角色：

# 列表项结构
| 信息 | 字段 | 位置 | 推断展示方式 | 格式化规则 |
|------|------|------|-------------|-----------|
| 主标题 | ___ | 左上 | 参照 §6.7 自动推断 | — |
| 副信息 | ___ | 左下 | 参照 §6.7 自动推断 | §6.6 格式化 |
| 状态 | ___ | 左下 | StatusTag 组件 | — |
| 金额 | ___ | 右侧 | 加粗格式化 | formatAmount() |
| 时间 | ___ | 副信息中 | 相对时间 | formatRelativeTime() |

# 列表项密度
- 标准（120rpx） / 紧凑（96rpx） / 富内容（auto，最小 140rpx）

# 搜索与筛选
- 搜索字段：（如：名称、编号）
- 筛选维度：（如：状态 [全部/待审批/已通过/已驳回]）
- 筛选方式：Filter Pill / 抽屉面板

# 操作按钮
- 按钮列表：（如：查看、审批、编辑、删除）
- 按钮数量：___
- 高频操作：___
- 危险操作：___
- 推荐模式：模式 A（内联+更多） / 模式 B（左滑 SwipeCell）

# 状态处理
- 空数据：EmptyState type=empty-data
- 搜索无结果：EmptyState type=no-search-results
- 加载失败：EmptyState type=load-failed
- 骨架屏：wd-skeleton，模拟 ___ 个列表项
- 加载更多：wd-loadmore

# 设计约束
- 遵循 `design-spec.md` § 6 列表设计规范
- 遵循 `design-spec.md` § 6.4 操作按钮规范
- 搜索栏吸顶
- 列表项间通过间距区分，不使用分割线
- 使用真实业务数据（如"采购合同-2024001"、"张三 · 采购部"），不用"标题"、"描述"
- 列表项内容应有差异，不要所有 item 完全相同
- 数据格式化：AI 根据字段名自动推断格式化方式，参照 `design-spec.md` §6.6 + §6.7
- 布局变体：AI 根据字段组成自动选择列表项布局，参照 `design-spec.md` §6.8
- 批量操作：如需多选功能，遵循 `design-spec.md` §6.9

# 输出要求
- 给出列表项的信息布局
- 给出搜索和筛选的位置与交互
- 给出操作按钮的模式选择及原因
- 给出各种状态的处理方式
```

## Detail Page Template (详情页面专用)

Use this template when the page specialty is `detail`.

```md
# 详情页面信息
- 页面名称：（如：合同详情 / 审批详情 / 用章记录）
- 业务目标：
- 目标角色：

# 页面头部（三层信息层级，参照 §7.5）
- 第一层（标题+状态）：___（StatusTag 紧跟标题）
- 第二层（关键指标）：___（如金额，加粗放大；无数值则省略此层）
- 第三层（元信息）：___（创建人 · 部门 · 时间）

# 信息分组
| 分组名 | 包含字段 | 展示方式 | 推断展示方式 |
|--------|----------|----------|-------------|
| 基本信息 | 字段1、字段2、字段3 | label-value 对 | 参照 §7.4 自动推断 |
| 签署信息 | 字段4、字段5 | label-value 对 | 参照 §7.4 自动推断 |
| 审批记录 | 审批人、时间、结果 | 时间线（wd-steps 竖向） | — |
| 附件 | 文件/图片列表 | 参照 §7.6 附件规范 | — |
| 关联数据 | 关联列表 | 参照 §7.7 关联数据规范 | — |

# 关键字段高亮
- 金额字段：使用 $font-weight-bold + $font-size-md
- 状态字段：使用 StatusTag + 四层状态色

# 底部操作栏
- 按钮数量：___
- 按钮列表：（如：驳回、通过）
- 主操作：___（$color-primary）
- 次操作：___
- 危险操作：___（$color-danger，需二次确认）

# 设计约束
- 遵循 `design-spec.md` § 7 详情页设计规范
- 使用 PageSection + PageCard 分组
- 底部操作栏固定定位 + safe-area-bottom
- 页面内容底部预留 120rpx 留白
- 信息分组应有主次之分：关键信息组靠前且视觉更突出
- 使用真实业务数据（如"合同金额 ¥125,000.00"、"甲方 XX科技有限公司"），不用"字段值"
- 字段推断：AI 根据字段名自动推断展示方式和交互行为，参照 `design-spec.md` §7.4
- 头部层级：详情页头部遵循三层信息层级规范，参照 `design-spec.md` §7.5
- 附件展示：图片/文件附件遵循 `design-spec.md` §7.6
- 关联数据：嵌套列表遵循 `design-spec.md` §7.7，最多展示 3 条

# 输出要求
- 给出信息分组和排列优先级
- 给出状态展示位置和样式
- 给出底部操作栏的按钮布局
- 给出时间线/审批记录的展示方式
```
